## 第一层：立即执行（30分钟）

### 1.1 搜索栏条件显示

**问题：** <30 条数据时搜索无意义，占用首屏空间。

**方案：**
```vue
<div v-if="diagnosesData.length > 30" class="fe-history__search">
  <van-search v-model="searchQuery" ... />
</div>
```

### 1.2 诊断 tab 顶部总结行

**问题：** 进入诊断 tab 看不到全局状态。

**方案：** 在搜索栏下方、卡片列表上方加一行摘要：
```vue
<div v-if="diagnosesData.length > 0" class="fe-history__diag-summary">
  <span>📊 {{ diagnosesData.length }}次诊断</span>
  <span>· 最新 <strong>{{ latestScore }}</strong>分 {{ latestGrade }}</span>
  <span>· 最高 <strong>{{ bestScore }}</strong>分</span>
</div>
```
新增 computed:
```typescript
const latestScore = computed(() => diagnosesData.value[0]?.overallScore ?? 0)
const latestGrade = computed(() => diagnosesData.value[0]?.grade ?? '')
const bestScore = computed(() => Math.max(...diagnosesData.value.map(d => d.overallScore), 0))
```

### 1.3 诊断卡片趋势箭头

**问题：** 每张卡片只显示分数，看不出相比上次是进步还是退步。

**方案：** 在与前一次诊断的分数差上加箭头：
```vue
<span v-if="getScoreDelta(d, idx) !== 0" class="fe-history__card-delta" :class="getScoreDelta(d, idx) > 0 ? 'is-up' : 'is-down'">
  {{ getScoreDelta(d, idx) > 0 ? '↑' : '↓' }}{{ Math.abs(getScoreDelta(d, idx)) }}
</span>
```
```typescript
function getScoreDelta(d: DiagnosisReport, idx: number): number {
  if (idx >= diagnosesData.value.length - 1) return 0
  const prev = diagnosesData.value[idx + 1]
  return d.overallScore - prev.overallScore
}
```

### 1.4 通知 tab → 合并到铃铛信号

**问题：** 通知 tab 只有「分析完成」一种内容，与 AppHeader 铃铛的信号系统是两套独立数据源。用户困惑。

**方案：** 
- 删除通知 tab
- Tab 精简为 2 个：诊断 / 修炼
- 诊断完成事件发送到 signalStore（统一走铃铛体系）
- 或者在通知 tab 空态时显示「通知已迁移到顶部铃铛，点击查看全部信号」

目标存储在 localStorage：
```typescript
const LOCAL_GOAL_KEY = 'vct_weekly_practice_goal'
const weeklyGoal = ref(Number(localStorage.getItem(LOCAL_GOAL_KEY)) || 3)
```

## 第三层：架构优化（后续迭代）

### 3.1 信号系统统一
- 诊断完成通知 → signalStore（不再走 IndexedDB notification）
- 修炼逾期提醒 → signalStore
- 删除通知 tab，铃铛成为唯一通知入口

### 3.2 修炼→复分析→对比 闭环
- 修炼完成后自动提示复分析
- 复分析完成后自动展示前后对比
- 对比报告中标注「本次提升来自修炼项X、Y」

### 3.3 缓存真实落地
- `POST /diagnoses/{id}/cache` 实际写入 Redis/文件
- 复分析时跳过 URL 采集，直接读缓存
- 30天自动过期
