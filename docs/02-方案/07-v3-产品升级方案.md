# 第一篇：AI 分析模块升级

## 1. 产品定位升级

**之前**：视频诊断工具 — 粘贴链接 → 打分 → 给建议。角色是**评判者**。

**之后**：创作者自我认知系统 — 输入任何内容 → 看见自己没看见的东西 → 产生新的自知。角色是**镜子**。

## 2. 价值模型

### 2.1 四个价值层次

| 层次 | 系统说什么 | 用户感受 |
|------|-----------|---------|
| 1 评判 | "这条视频 72 分" | 哦，知道了 |
| 2 揭示 | "第 7 秒流失 40%，hook 承诺没兑现" | 原来问题出在这里 |
| 3 关联 | "最近 5 条全在第 5-10 秒出现同样拐点，你的 hook 有系统性问题" | 这是我自己的模式 |
| 4 自知 | "你擅长讲故事（85），但总在硬塞干货（52）。你不是干货型博主，是故事型博主" | 这个产品懂我 |

### 2.2 三个价值转化

- **绝对值→相对值**："你 65 分，但你自己去年 11 月那条到过 82。你曾经做到过。"
- **发现→自我认知**："第 8 次变现预埋低于 50 分。不是你做不到，是你潜意识觉得谈钱伤粉。"
- **知道→做到**："把每句话从 2.5 秒剪到 1.2 秒。对标 TOP3 平均就是 1.2 秒。下条试试，涨了回来告诉我。"

### 2.3 场景→模式映射

| 用户想… | 匹配模式 |
|----------|---------|
| "这视频行不行？" | 快速扫描（30s） |
| "怎么改最好？" | 深度分析 |
| "他为什么比我火？" | 对比分析 |
| "这些爆款有什么共性？" | 矩阵分析 |
| "赛道在往哪走？" | 趋势识别 |
| "先改哪个 ROI 最高？" | 差距定位 |
| 不想选 | 智能推荐（默认） |

## 3. 动态因子系统

### 3.1 三条固定参照线

```
90 ┤ ─ ─ ─ ─ ─ 头部线（赛道 TOP 5%）
80 ┤ ─ ─ ─ ─ ─ 专业线（可持续变现基线）
70 ┤ ─ ─ ─ ─ ─ 及格线（平台基础分发条件）
   │
60 ┤  ●───●───●  你的轨迹 →
50 ┤●
   └┼───┼───┼───┼───┼───
```

### 3.2 七因子自动权重

| # | 因子 | 做什么 | 更新频率 | 范围 |
|---|------|--------|---------|------|
| 1 | 样本成熟度 | 样本不足时压权重，逐步释放 | 每周 | 0.2–1.0 |
| 2 | 时效衰减 | 旧数据指数衰减 | 每日 | 0.15–1.0 |
| 3 | 区分度 | 方差极小 → 降权 | 每 5000 样本 | 0.3–1.5 |
| 4 | 因果关联 | 与播放/互动 r 驱动权重 | 每 10000 验证 | 0.3–1.5 |
| 5 | 平台变动 | 检测算法更新 | 每周 | — |
| 6 | 采纳反馈 | 建议被采纳且有效 → 加权重 | 每月 | 0.7–1.3 |
| 7 | 赛道饱和 | 某赛道某维同质 → 降权 | 每月 | 0.3–1.2 |

# 第三篇：跨模块桥接闭环

## 9. 四条桥接路径

```
         ┌──────────────────────────────────┐
         │          火眼金睛 工作流           │
         │                                  │
         │  AI分析 ──→ 发现弱项              │
         │    │        ↓                    │
         │    │    WeaknessToGeneCard       │
         │    │        ↓                    │
         │    │     找对标 ──→ 改善模拟      │
         │    │        ↓                    │
         │    │     改进后重新分析            │
         │    │        ↓                    │
         │    └──→ 诊断对比 ←── 历史         │
         │              ↓                   │
         │          成长卡片                 │
         └──────────────────────────────────┘
```

### 桥接 1：诊断 → 对标（已有 WeaknessToGeneCard，需修复）

**当前**：DiagnosisPage 调用 `cacheStore.buildDiagnosisContext()` 但结果被丢弃；GenePage 自己重新从 IndexedDB 加载。

**修复**：DiagnosisPage 直接将 context 写入 `geneStore.diagnosisContext`，GenePage 不再独立查询。

### 桥接 2：对标 → 诊断（需新建）

**当前**：不存在。对标出结果后用户无法直接分析自己的视频。

**新增**：GenePage 的「用此对标分析我的视频」按钮 → 跳转 DiagnosisPage 并预填对标链接。

### 桥接 3：诊断 → 历史 → 对比（部分已有）

**当前**：复分析绕过 taskStore，无进度追踪。

**修复**：HistoryPage 复分析走 `taskStore.createTask()` 管道。

### 桥接 4：全部 → 成长档案（已有 GrowthCardStore，需补充分享）

**当前**：分享按钮 handler 为空。

**修复**：HomePage / LeftDrawer 接上 `shareSlug` 生成分享链接。

## 10. 左右抽屉配合规则

| 场景 | 左抽屉 | 主区 | 右抽屉 |
|------|--------|------|--------|
| 用户打开左抽屉点「AI分析」 | 关闭 | 跳转诊断页 | 保持当前上下文 |
| 分析提交后 | 推荐区更新 | 显示进度 | 自动切到 `diagnosis:progress` |
| 分析完成 | 推荐区出现「查看报告」 | 显示 WeaknessToGeneCard | 自动切到 `diagnosis:report` |
| 用户点「找对标」 | 关闭 | 跳转 gene 页 | 切到 `gene:idle` |
| 对标出结果 | 推荐区出现「分析我的视频」 | 显示因子报告 | 切到 `gene:factor_report` |

实现关键：`drawerStore.setRightCtx()` 已在关键节点可调用。
