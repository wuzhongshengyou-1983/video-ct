精度分层规则（precision_tier）
精度层级触发条件示例
track_exact match_level = "exact" 且 sub_dimensions ≥ 2 个 「口播」+ 内容类型:观点输出 + 时长:<30秒 → 赛道专属分析
track_nearby match_level = "exact" (0-1 子维度) 或 match_level = "alias" 且 sub_dimensions ≥ 3 「口播」（无子维度）→ 赛道近似 / 「舞蹈」+ 3个子维度 → 近似赛道
platform_general 其他情况（别名 + 少量子维度 / 只有签名推断） 「手工」（仅推断）→ 平台通用基准

决策矩阵：做了 vs 没做
决策理由
✅ 已做 别名映射扩展零风险，立即覆盖 23 个品类
✅ 已做 三级品类匹配用户知道为什么被匹配到某个赛道
✅ 已做 点击追踪 + 热门排序数据驱动自动调节，不需要人工维护
✅ 已做 子维度全面扩展23 品类 × 3-4 子维度 = 92 个子维度
✅ 已做 precision_tier 联动子维度数量影响精度标签，用户感知明显
⏸ 待做 子维度→基准池过滤需要 TikHub 数据积累和规则打标签
⏸ 待做 子维度→方法论排序需要为每个 content_type 打方法论标签
❌ 不做 子维度→权重修正（现在）384 个猜测值会制造精确的假象，等有数据再说
❌ 不做 新增核心赛道（现在）没有基准数据的新赛道 = 随机结果
❌ 不做 重构 factor_calculator架构已锁定，计算公式不变
