# 火眼金睛 AI 分析 — 视觉验收与修复方案 v4

## 概述

基于 v3 升级完成后的全模块人工视觉检查，发现 4 类问题：
1. 四种分析模式的展现方式存在语义重叠、退化、元数据缺失
2. 代码级 Bug（随机数污染、类型非法值、虚假映射）
3. Phase 3/4 新增组件的数据管道断裂（2/3 不可用）
4. 全局视觉一致性与状态覆盖缺失

---

## 一、四种分析模式展现问题

### 1.1 智能推荐 — 缺少透明度

**现状**：ModeSelector 将其作为同级芯片呈现，实际是一个 meta 代理 — 静默委托到其他模式。

**问题**：
- 用户不知道「为什么推荐这个模式」
- 没有任何推荐理由或策略说明
- 选中态视觉和实际分析模式完全一样

**方案**：
- 移除「智能推荐」作为同级芯片
- 当用户选择智能推荐时，在最终生效的模式芯片上添加「🤖 推荐」角标
- 在输入卡下方增加一行 11px 灰色说明文字：「根据 3 条链接+赛道，推荐矩阵分析」
- ModeSelector 新增 `recommendReason` prop 展示推荐理由

**影响文件**：
- `apps/h5/src/components/analysis/ModeSelector.vue` — 新增 recommendReason prop，推荐角标样式
- `apps/h5/src/views/fire-eye/DiagnosisPage.vue` — suggestedMode 返回值对齐 AnalysisMode 类型

---

### 1.2 对比分析 vs 差距定位 — 语义重叠

**现状**：

| 模式 | 描述 | 实际行为 |
|------|------|----------|
| 对比分析 | 「我的 vs 对标 全维度差距」 | 多链接两两对比 |
| 差距定位 | 「精准定位薄弱环节」 | 我的 vs 头部单点差距 |

DiffReport 和 CompareReport 两个组件视觉结构几乎相同（分数 hero + 维度条形图），但独立维护。

**问题**：
- 「对比分析」输入层指多链接对比，输出层 CompareReport 指修炼前后对比 — 同名异义
- DiffReport 和 CompareReport 维护成本翻倍
- difference 阈值不统一：CompareReport 用 ±3，DiffReport 不设阈值

**方案**：
- 合并 DiffReport 和 CompareReport 为一个 `ComparisonReport` 组件
- 通过 `type: 'temporal' | 'benchmark'` prop 区分场景
- 统一 delta 阈值（±3 分视为有意义变化）
- 更新所有引用处

**影响文件**：
- `apps/h5/src/components/fire-eye/CompareReport.vue` → 重构为 ComparisonReport
- `apps/h5/src/components/fire-eye/DiffReport.vue` → 删除，功能合并
- `apps/h5/src/components/fire-eye/DiagnosisReport.vue` → 更新引用
- `apps/h5/src/views/fire-eye/HistoryPage.vue` → 更新引用
- `apps/h5/src/components/fire-eye/HistoryRightContent.vue` → 更新引用

---

### 1.3 矩阵分析 — 5+ 列退化

**现状**：<=4 列显示热力图（交叉表格），>=5 列降级为卡片列表。

**问题**：
- 卡片列表丧失了矩阵的「交叉对比」核心价值 — 用户无法在同一行比较多个视频的同一维度得分
- 行标签 56px 宽度在五字中文维度名时溢出
- 热力颜色使用硬编码 RGB 而非 CSS 变量

**方案**：
- >=5 列时改为横向可滚动的紧凑表格（保持交叉表结构）
- 固定第一列（维度名）+ 固定「我的」列，其余可横向滚动
- 行标签宽度改为自适应，最小 64px
- 热力颜色改用 CSS 变量 `var(--vct-primary)` 系列
- 新增 hover 高亮整行 + 整列

**影响文件**：
- `apps/h5/src/components/analysis/CompareMatrix.vue`

---

### 1.4 趋势识别 — 单线图表过于基础

**现状**：SVG 折线图，硬编码 viewBox 340×200，仅支持单条折线。

**问题**：
- viewBox 不响应容器宽度
- 仅支持单条折线，无法叠加多条趋势对比
- Y 轴固定 0-100
- 无趋势方向指示（上升/下降摘要）
- 无异常点高亮

**方案**：
- viewBox 改为动态计算（基于容器宽度）
- 新增 `series` prop 支持多条折线叠加
- Y 轴范围自动适配数据（min/max 取整到 5 的倍数）
- 图表顶部新增趋势摘要条（「整体上升 +12%」「波动较大」等）
- 异常点（偏离均值 2σ）用红色圆环标记

**影响文件**：
- `apps/h5/src/components/analysis/TrendLine.vue`

## 二、代码级 Bug 修复

### 2.1 DiagnosisReport — computed 中 Math.random()

**位置**：`DiagnosisReport.vue:556`

```diff
- : 0.65 + Math.random() * 0.15,
+ : 0.7,
```

computed 中不应使用随机数，改为固定默认置信度 0.7。

---

### 2.2 DiagnosisPage — suggestedMode 返回非法值 'auto'

**位置**：`DiagnosisPage.vue:250-261`

`AnalysisMode` 类型无 `'auto'`，但 `suggestedMode` 在两处返回 `'auto'`。应改为 `'smart_recommend'`。

```diff
- if (n <= 1) return 'auto'
+ if (n <= 1) return 'smart_recommend'
```

同时 watch 中不再需要 `as AnalysisMode` 强制转换。

---

### 2.3 DiagnosisReport — 百分位虚假映射

**位置**：`DiagnosisReport.vue:500-508`

移除硬编码映射，改为仅在有真实基准数据时显示：

```diff
- const percentileText = computed(() => {
-   if (!benchmark.value) return '...'
-   const gap = benchmark.value.overallGapPct
-   if (gap >= 10) return '90%'
-   ...
- })
+ const percentileText = computed(() => {
+   if (!benchmark.value?.percentile) return null
+   return `${benchmark.value.percentile}%`
+ })
```

并在模板中改为 `v-if="percentileText"` 条件渲染（null 时不显示该区块）。

## 三、数据管道修复

### 3.1 DiffReport / ComparisonReport 数据断连

**问题**：`compareData` 从 `reportData.value?.compare` 读取，但该字段从未被写入。

**方案**：
- 后端 `diagnosis_service.py` 在生成报告时写入 `compare` 字段（当存在历史诊断时自动计算维度差）
- 前端 `switchToReport` / `openHistReport` 在缓存写入前，从 IndexedDB 查找上一次诊断并组装 compare 数据
- 模板改为 `v-if="compareData"` 已有，无需修改

**影响文件**：
- `services/api/app/services/diagnosis_service.py` — 计算 compare 数据
- `apps/h5/src/views/fire-eye/DiagnosisPage.vue` — switchToReport 组装 compare

### 3.2 DataSourcePanel 数据断连

**问题**：`sourceTransparency` 从 `reportData?.report?.source_transparency` 读取，但该字段从未被填充。

**方案**：
- 后端 `diagnosis_service.py` 在报告 JSON 中写入 `source_transparency` 对象
- 结构：`{ sources: {...}, overall_quality_score: 0.8, quality_label: '标准诊断', disclaimer: '...' }`
- 各 agent 在运行时通过 `data_source_bus.py` 上报实际使用的数据源
- 前端 `switchToReport` 透传该字段

**影响文件**：
- `services/api/app/services/diagnosis_service.py` — 填充 source_transparency
- `services/api/app/services/data_source_bus.py` — 暴露数据源摘要接口

## 四、全局视觉一致性修复

| 类别 | 当前 | 目标 |
|------|------|------|
| 成功色 | `#34d399` 硬编码在多处 | 统一为 `var(--vct-success)` |
| 危险色 | `#f87171` 硬编码在多处 | 统一为 `var(--vct-danger)` |
| 文字色 | `#4CAF50`, `#F5A623` 等硬编码 | 统一用 `var(--vct-success)`, `var(--vct-warning)` |
| 字号 | `font-size: 11px/10px/9px` 硬编码 | 统一为 `var(--vct-fluid-text-*)` |
| 圆角 | `var(--vct-radius)` / `--sm` / `--md` 混用 | 卡片用 `--md`，内部元素用 `--sm` |

**影响文件**：
- `DiffReport.vue` → ComparisonReport 重构时一并修复
- `CompareReport.vue` → 同上
- `TrendLine.vue`
- `CompareMatrix.vue`

## 五、状态覆盖补全

| 组件 | 缺失 | 方案 |

| ModeSelector | 不适用模式的禁用态 | 单链接时「矩阵」「趋势」置灰 + tooltip「需要 2+ 条链接」 |
