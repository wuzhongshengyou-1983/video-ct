## 8. Phase-1 极速诊断前端集成（2026-05-20）

**背景**：Python 端 `/api/fire-eye/v2/diagnose-fast` (9.4s，≤20s) 已就绪，前端仍走老 `/api/fire-eye/diagnose` (60-90s)。

**前端改造**（`ruyi-h5/index.html`）：
- **阶段UI**：诊断按钮触发后改为 2 阶段（提取真实数据 + 文案+五大曲线），标题显示「⚡ 极速诊断 Phase-1 · 目标 ≤20秒」
- **新增 `renderStages()`**：动态渲染阶段列表，被 fast/fallback/demo 三路径复用
- **重写 `startSSE()`**：优先调 `/api/fire-eye/v2/diagnose-fast`（Phase-1），解析 SSE 事件（start/progress/complete/error），complete 后自动 `launchPhase2()`
- **新增 `fallbackToFullDiagnose()`**：fast 端点不可用（503等）或数据提取失败（event: error）时，自动降级重建 5 阶段 UI + 调老端点
- **新增 `launchPhase2()`**：Phase-1 完成后后台调老端点 LLM 深度分析，结果回填 `deep_reasoning`/`root_cause`/`groups`/`total_score`，静默更新 hero 摘要，结果标题加 「🧠 AI已深化」
- **`onDiagnoseComplete()`**：阶段完成标记改为动态（`querySelectorAll('#stageList .stage-row')`），适配 2/5/6 阶段

**降级链**：fast → (event:error or 503) → fallbackToFullDiagnose → (fail) → runDemoMode

**验证**：
- `curl` fast 端点通过 8700 代理 → 9.4s 返回完整报告（5曲线 + 文案 + gene_alignment + data_source:tikhub_realtime）
- JS 语法检查通过（所有 script 块 `new Function()` 解析无异常）
- 浏览器加载无 JS 错误，全部新函数（renderStages/startSSE/fallbackToFullDiagnose/launchPhase2）可用
