## 一、全局架构观

### 1.1 产品定位

火眼金睛 H5 = **短视频从业者的参考+诊断工具**。
两条服务路径：
- 「找参考」— 给我看同行在做什么（基因链路）
- 「诊断我」— 给我分析我的视频哪里有问题（诊断链路）

### 1.2 设计三角

```
          极简表面
         /        \
  专业深度  ——  两条独立链路
```

表面只有两个选择、四个Tab、两个抽屉；内容深度和数据质量是护城河。

### 1.3 整体视图

### 1.4 五大设计原则

| # | 原则 | 展开说明 |
|---|------|----------|
| P1 | 先只做 H5 | 移动端浏览器优先，小程序/App 对齐在 Phase 7 |
| P2 | 简单方便 | 能删的删、能合的合、零文字输入（除粘贴链接） |
| P3 | 选项用抽屉 | 所有多选项展示用 BottomSheet 或 SideDrawer，不平铺 |
| P4 | 双抽屉常驻 | 左右抽屉随时可调，右抽屉内容随页面切换 |
| P5 | 内容专业 | 表面入口少，诊断深度/推荐质量不打折 |

## 四、应用状态机

### 4.1 全局 App 状态

```
AppState {
  currentTab: 'home' | 'gene' | 'diagnosis' | 'history'
  leftDrawer: { open: boolean }
  rightDrawer: { open: boolean; context: DrawerContext }
  notifications: Notification[]
  backgroundTasks: Task[]
}

DrawerContext = HomeContext | GeneContext | DiagnosisContext | HistoryContext
```

### 4.2 页面保活策略

```
页面切换时：
  - 主页：不保活（轻量入口，每次重渲染）
  - 基因：保活选择状态（平台/阶段/赛道/维度）
  - 诊断：保活任务 ID（Worker 在后台继续运行）
  - 历史：不保活（从 IndexedDB 重新加载）

技术实现：React Router keepAlive（display:none 隐藏，不卸载）
```

### 4.3 后台任务状态机

```
TaskState: pending → extracting → analyzing → completed | failed

pending     — 任务创建，等待 Worker 空闲
extracting  — 数据抓取中（平台 API 调用）
analyzing   — AI 四维分析中
completed   — 结果写入 IndexedDB，通知用户
failed      — 错误已记录，支持重试
```

│  粘贴视频链接                    │

**四维诊断详细规格**：

| 维度 | 分析内容 | 数据来源 | 输出格式 |
|------|---------|---------|---------|
| 文案 | 标题质量/字幕密度/关键词覆盖/CTA强度/情绪词频 | 视频字幕提取+标题抓取 | 评分+具体改进建议（5条） |
| 视频 | 前3秒钩子/画面稳定性/构图/切换节奏/字幕覆盖时长 | 帧抽样分析 | 时间轴标注+评分 |
| 音频 | 语速节奏/音量波动/BGM/静音段分布/声音清晰度 | 音轨分析 | 波形摘要+问题点 |
| 数据 | 播放量/完播率/点赞率/评论率/分享率/涨粉率 | 平台公开数据+TikHub | 指标卡+同类对比 |

**诊断交互流**：

```
1. 粘贴链接 → 自动识别平台
2. 右抽屉弹出：【数据提取中】进度条

3. 数据整理完成 → 右抽屉切换：【选择诊断维度】

4. 用户选择维度 → 右抽屉：【AI分析中】
5. 分析完成 → 右抽屉：完整诊断报告
   └── 分段展示各维度结果
   └── 「保存报告」→ 存入历史
6. 用户切换页面 → 后台继续，状态条可见
```

## 七、两条核心链路（完整数据流）

### 7.1 链路一：基因链路

```
[用户操作]
   ↓
主页「找参考」按钮 / Tab直接点击
   ↓
[基因页] 展示四级选择器引导
   ↓
右抽屉弹出「平台选择」BottomSheet
   → 用户选择平台（如：抖音）
   ↓
「账号阶段」BottomSheet
   → 用户选择（如：成长期）
   ↓
「赛道选择」BottomSheet（动态列表）
   → API: GET /gene/categories?platform=douyin
   → 用户选择（如：美食·家常菜）
   ↓
「诊断维度」BottomSheet（多选）
   → 用户选择（如：文案+选题）
   ↓
[确认] → API: POST /gene/recommend
        { platform, stage, category, dimensions }
   ↓
右抽屉：骨架屏加载（最长 3s）
   ↓
API 返回推荐视频列表（10~20条）
右抽屉：渲染视频卡片
   → 封面 / 标题 / 平台数据 / 维度标签
   ↓
用户点击某视频卡片
   → 右抽屉切换到详情视图
   → 展示：视频预览 / 完整数据 / 维度分析摘要
   ↓
[加入参考库] 按钮
   → IndexedDB.geneRecords.add(record)
   → Toast「已加入参考库」
   ↓
历史页「基因记录」分区可见
```

**数据持久化节点**：
- 四级选择条件：存 sessionStorage（保活）
- 推荐结果：存 IndexedDB（长期）
- 收藏的单条视频：存 IndexedDB

---

### 7.2 链路二：诊断链路

```
[用户操作]
   ↓
主页「诊断我」按钮 / Tab直接点击
   ↓
[诊断页] 展示粘贴输入区
   ↓
用户粘贴链接（如：抖音分享链接）
   → 自动识别平台 + 提取视频ID
   → 链接格式校验
   ↓
[API: POST /task/create]
   → 返回 taskId
   → IndexedDB.tasks.add({ id, status: 'pending', url, createdAt })
   ↓
[Worker 启动]
Phase 1: 数据提取
   → API: GET /extract?taskId={id}
   → 平台数据（播放/点赞/完播率等）
   → 字幕提取
   → 封面/基础信息
   → Worker → BroadcastChannel → 主线程
   → 右抽屉更新进度（step 1/4）
   ↓
Phase 2: AI 四维整理
   → 并行分析（文案/视频/音频/数据）
   → 每完成一维，右抽屉进度更新
   ↓
[用户切换 Tab（可能发生）]
   → Worker 不中断，继续运行
   → 状态条（主页/Header）更新进度
   ↓
Phase 3: 生成报告
   → AI 汇总四维结果
   → 生成结构化 JSON 报告
   → 写入 IndexedDB.diagnoses
   → IndexedDB.tasks.update(status: 'completed')
   ↓
[通知触发]
   → BroadcastChannel 广播完成事件
   → 若在主页：状态条变绿 ✓
   → 若在其他页：Header 铃铛角标 +1
   → 若在诊断页：右抽屉自动切到结果视图
   ↓
[用户查看结果]
右抽屉展示完整报告：
   ┌─ 视频基本信息
   ├─ 数据总览（雷达图）
   ├─ 文案诊断（折叠展开）
   ├─ 视频诊断（折叠展开）
   ├─ 音频诊断（折叠展开）
   └─ 数据诊断（折叠展开）
   ↓
[保存/分享]
   → 「加入修炼」→ 历史页修炼中分区
   → 「复制摘要」→ 剪贴板
```

**后台任务保活关键**：
- Worker 通过 taskId 持续轮询任务状态（不依赖页面是否可见）
- visibilitychange 为 hidden 时降低轮询频率（30s/次）
- visibilitychange 为 visible 时立即检查状态并推送更新

## 九、数据层设计（IndexedDB）

### 9.1 数据库结构

数据库名：`fire-eye-db`，版本：`3`

```
stores:
  tasks          — 后台任务记录
  diagnoses      — 诊断报告
  geneRecords    — 基因搜索+推荐记录
  geneVideos     — 推荐视频详情（可被多条 geneRecord 引用）
  practices      — 修炼中条目
  notifications  — 通知记录（已读/未读）
```

## 十、后台任务系统

### 10.1 Worker 架构

```
主线程                          Worker 线程
   │                                │
   ├─ postMessage(CREATE_TASK)  ──→ │
   │                                ├─ 数据提取（axios）
   │                                ├─ AI分析请求
   │ ←── BroadcastChannel(PROGRESS) ┤
   │ ←── BroadcastChannel(COMPLETE) ┤
   │                                ├─ 写入 IndexedDB（Worker可直接访问）
   └─────────────────────────────── ┘
```

### 10.2 并发控制

- 最大并发任务数：**2**（超出进入队列）
- 任务优先级：最新创建的任务优先
- Worker 重启策略：崩溃后自动重启，任务从 IndexedDB 恢复 pending 状态

### 10.3 任务恢复

```
App 启动时：
  1. 查询 IndexedDB.tasks WHERE status IN ('extracting', 'analyzing')
  2. 这些任务状态重置为 'pending'
  3. 重新推入 Worker 队列
  4. （上次进度丢失，从头开始；不影响用户，因为结果还没出）
```

## 十一、通知系统

### 11.1 通知类型

| 类型 | 触发时机 | 显示位置 | 持续时间 |
|------|---------|---------|---------|
| TASK_COMPLETE | 诊断任务完成 | Header 铃铛 + Toast | 铃铛保留到用户点击 |
| TASK_FAILED | 任务失败 | Toast + 铃铛（红色） | — |
| GENE_LOADED | 基因推荐加载完成 | 右抽屉内自动切换 | — |
| DATA_SAVED | 收藏/保存成功 | Toast（底部） | 2s |

### 11.2 铃铛徽标规则

- 未读通知数 > 0 → 显示数字
- 点击铃铛 → 展开通知中心（BottomSheet）
- 点击单条通知 → 跳转对应页面 + 标记已读

## 十二、错误处理策略

### 12.1 网络错误

| 场景 | 处理方式 |
|------|---------|
| 基因推荐 API 失败 | Toast 提示 + 重试按钮（右抽屉内） |
| 数据提取失败 | 任务状态 → failed，右抽屉显示原因 + 重试 |
| AI 分析超时（>60s） | 继续等待，提示「AI 正在努力分析」|
| AI 分析失败 | 已提取的原始数据仍保存，部分维度标记「分析失败」|
| 链接格式错误 | 即时校验，inline 错误文字（不弹 Sheet）|
| 不支持的平台 | 友好提示支持的平台列表 |

### 12.2 离线降级

| 功能 | 离线状态 |
|------|---------|
| 历史页 | 完全可用（IndexedDB） |
| 诊断页 | 禁用粘贴，提示需要网络 |
| 基因页 | 禁用选择器，提示需要网络 |
| 主页 | 显示本地历史，新任务入口禁用 |
