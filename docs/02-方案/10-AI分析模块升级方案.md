# 第一篇：为什么做

## 1. 产品定位升级

**之前**：视频诊断工具 — 粘贴链接 → 打分 → 给建议。角色是**评判者**。

**之后**：创作者自我认知系统 — 输入任何内容 → 看见自己没看见的东西 → 产生新的自知。角色是**镜子**。

## 2. 价值模型

### 2.1 四个价值层次

| 层次 | 系统说什么 | 用户感受 |
|------|-----------|---------|
| 1 评判 | "这条视频 72 分" | 哦，知道了 |
| 2 揭示 | "第 7 秒流失 40%，hook 承诺没兑现" | 原来问题出在这里 |
| 3 关联 | "最近 5 条全在第 5-10 秒出现同样拐点，你的 hook 有系统性问题" | 这是我自己的模式 |
| 4 自知 | "你擅长讲故事（85），但总在硬塞干货（52）。你不是干货型博主，是故事型博主" | 这个产品懂我 |

层次 4 是护城河——需要持续积累用户自己的数据，竞品抄不走。

### 2.2 三个价值转化

**绝对值→相对值**："你 65 分，但你自己去年 11 月那条到过 82。你曾经做到过。"

**发现→自我认知**："第 8 次变现预埋低于 50 分。不是你做不到，是你潜意识觉得谈钱伤粉。"

**知道→做到**："把每句话从 2.5 秒剪到 1.2 秒。对标 TOP3 平均就是 1.2 秒。下条试试，涨了回来告诉我。"

### 2.3 场景→模式映射

| 用户想… | 匹配模式 |
|----------|---------|
| "这视频行不行？" | 快速扫描（30s） |
| "怎么改最好？" | 深度分析 |
| "他为什么比我火？" | 对比分析 |
| "这些爆款有什么共性？" | 矩阵分析 |
| "赛道在往哪走？" | 趋势识别 |
| "先改哪个 ROI 最高？" | 差距定位 |
| 不想选 | 智能推荐（默认） |

# 第二篇：核心机制

## 3. 标准体系

### 3.1 标准的三类来源

**统计因果**：前 3 秒完播率与最终完播率 r=0.73（不是感觉，是相关��数）

**平台规则**：抖音 2025.9 更新后互动率权重 35%→52%，纯完播策略正在失效

**生理下限**：字幕 <14px 阅读速度降 40%；单镜头 >8s 无变化，注意力流失每秒 +12%

### 3.2 三条固定参照线

```
90 ┤ ─ ─ ─ ─ ─ 头部线（赛道 TOP 5%）
80 ┤ ─ ─ ─ ─ ─ 专业线（可持续变现基线）
70 ┤ ─ ─ ─ ─ ─ 及格线（平台基础分发条件）
   │
60 ┤  ●───●───●  你的轨迹 →
50 ┤●
   └┼───┼───┼───┼───┼───
```

三条线位置固定。你的轨迹在线的参照下才有意义。

### 3.3 全自动动态因子系统

标准线固定，但**维度权重**由 7 个动态因子自动调节，无人介入：

| # | 因子 | 做什么 | 更新频率 | 范围 |
|---|------|--------|---------|------|
| 1 | 样本成熟度 | 样本不足时压权重，逐步释放 | 每周 | 0.2–1.0 |
| 2 | 时效衰减 | 旧数据指数衰减，新数据优先 | 每日 | 0.15–1.0 |
| 3 | 区分度 | 方差极小（大家都一样）→ 降权 | 每 5000 样本 | 0.3–1.5 |
| 4 | 因果关联 | 与真实表现（播放/互动）r 驱动权重 | 每 10000 验证 | 0.3–1.5 |
| 5 | 平台变动 | 检测算法更新，加速数据刷新 | 每周 | — |
| 6 | 采纳反馈 | 建议被采纳且有效 → 加权重 | 每月 | 0.7–1.3 |
| 7 | 赛道饱和 | 某赛道某维同质 → 降权 | 每月 | 0.3–1.2 |

**聚合规则**：
1. 各因子相乘 → raw_weight
2. clamp 到 `[base×0.2, base×3.0]`（防极端）
3. 全部维度归一化至总和 1.0
4. 单维度保底不低于基准的 20%
5. 自动打版本号 `v{date}.{seq}`，全部可追溯

平台变动因子特有：触发前需排除样本量 <500 的噪声 + 其他平台同期同向的全行业趋势。

## 4. 动态系统

### 4.1 三层动态

**对你的记忆**：`creator_profile` 持续更新，每次分析锚定到你的历史曲线。新用户无数据时降级为静态模式，第 3 条分析后差分报告和趋势线自动激活。

**对赛道的感知**：对标是赛道的移动平均，不是固定快照。每次打开报告重新拉取。

**系统的进化**：方差极低的维度自动降权，与爆款高相关的模式自动引入。

### 4.2 建议采纳自动追踪

不需要用户反馈。下次分析新视频时自动推断：

```
上次建议：把节奏从 2.5s 缩到 1.2s
本次实测：节奏 1.4s
→ 系统判定：部分采纳
→ 该建议所属维度的采纳反馈因子自动计入
```

新增 `services/advice_tracker.py` 负责此逻辑。

### 4.3 冷启动策略

| 缺失数据 | 降级方案 |
|----------|---------|
| 无 profile | 首次分析使用静态报告，"这是你的第一条分析" |
| 无历史趋势 | 趋势线替换为"再分析 2 条开启趋势追踪" |
| 无采纳数据 | 建议正常给，标注"下次分析自动检查采纳" |
| 赛道样本 <100 | 标注"置信度较低"，样本成熟度因子自动压到 0.3 |

# 第三篇：交互设计

## 5. 输入层

### 5.1 交互原型

```
┌──────────────────────────────────────────────────┐
│  🔗  粘贴链接，AI 智能分析                          │
│                                                   │
│  ┌───────────────────────────────────────────────┐│
│  │ 🟢 我的  douyin.com/video/xxxxx          [×]  ││
│  │ 🔵 对标  bilibili.com/video/yyyyy        [×]  ││
│  │ 🔵 对标  xiaohongshu.com/note/zzzzz      [×]  ││
│  │                                               ││
│  │ ─── 粘贴或输入链接，每行一个 ───                ││
│  │ + 添加更多链接                                 ││
│  └───────────────────────────────────────────────┘│
│                                                   │
│  🧠 已识别                                        │
│  我的：抖音视频 ×1  对标：B站视频 ×1  小红书图文 ×1  │
│  共 3 条 · 同类平台 · 建议：差距定位               │
│                                                   │
│  ┌──────────┬──────────┐  32px 分段控件            │
│  │  ⚡ 快速   │  🔬 深度  │                          │
│  └──────────┴──────────┘                          │
│                                                   │
│  模式：[智能推荐✓] [对比] [矩阵] [趋势] [差距]       │
│                                                   │
│  [ 开始智能分析 3 条内容 ]   预计 90-180s · 3 配额   │
└──────────────────────────────────────────────────┘
```

视觉权重：输入区 70% → 识别反馈 15% → 模式控件 10% → 执行按钮 5%

### 5.2 快速/深度分段控件

160×32px，13px 字号。放在输入区和识别反馈下方，视觉权重很低。默认选中「深度」。

类比：Google 不让你选搜索深度，它默认给最好结果。

### 5.3 链接角色标记

每条链接 chip 左侧有 🟢我的 / 🔵对标 切换。默认第一条为「我的」。切换角色时智能推荐自动调整。角色信息存入 `link_labels`。

### 5.4 内容类型自动识别

```
URL 输入 → 前端正则预判（即时 chip）
        → 后端 HEAD 请求（title / OG / Content-Type）
        → 返回 content_type + 可用维度
        → 更新 chip + 触发模式推荐
```

| URL 模式 | 识别为 | 分析策略 |
|----------|--------|---------|
| 抖音/B站/快手视频 | 短视频 | 六维视频诊断 |
| 小红书笔记 | 图文笔记 | 四维图文诊断 |
| 知乎问答/文章 | 问答/文章 | 内容结构分析 |
| 公众号文章 | 公众号 | 传播力分析 |
| 电商详情页 | 电商产品 | 转化力分析 |
| 其他 | 通用网页 | 基础内容分析 |

### 5.5 多链接规则

- 1 条 → 单条分析
- 2-3 条 → 建议对比
- 4-10 条 → 建议矩阵/趋势
- 10+ → 提示分批，上限 10 条
- 任意 URL 可输入，无效链接红色标记
- 30 天内已分析过的 URL 提示复用，仅消耗 0.1 配额

### 5.6 首次引导

不需要引导页——输入框 placeholder 示例就是最好的引导，识别反馈条让用户立刻理解价值。

## 6. 分析模式

| 模式 | 输入 | 回答 | 核心输出 |
|------|------|------|---------|
| 快速扫描 | 1 条 | 这视频行不行？ | 总分+等级+3 发现（30s） |
| 深度分析 | 1 条 | 怎么改最好？ | 完整六维+对标+改善杠杆 |
| 对比分析 | 2-3 条同类 | 他为什么比我火？ | 并排对比+差距高亮+追赶建议 |
| 矩阵分析 | 3-6 条同类 | 共性是什么？ | N×6 热力图+聚类+最佳实践 |
| 趋势识别 | 4-10 条时间跨度 | 赛道往哪走？ | 时序趋势+阶段判断+机会窗口 |
| 差距定位 | 1 我+N 对标 | 先改哪个？ | GAP 排序+杠杆+分步路线 |
| 智能推荐 | 任意 | 不用选 | 自动匹配最优 |

### 智能推荐决策

| 输入特征 | 推荐 |
|----------|------|
| 1 条 | 深度分析 |
| 2 条同平台同类型 | 对比分析 |
| 2 条不同平台/类型 | 对比分析（跨平台） |
| 3-6 条同平台同类型 | 矩阵分析 |
| 3-6 条混合 | 矩阵分析+分类标记 |
| 4+ 条同创作者 | 趋势识别 |
| 我的+N 对标 | 差距定位 |
| 全是对标 | 矩阵分析（纯竞品研究） |
| 混合类型 | 分类分析+汇总简报 |
| fallthrough | 深度分析（保守） |

## 7. 输出层

### 7.1 单链接报告（默认差分视图）

顶部：总分数 + 趋势线 + 相比上次的变化（差分）

中部：六维详情（可折叠）+ 改善杠杆排序

底部：行动建议（本周/本月/本季）

有历史 → 默认差分视图。无历史 → 默认绝对视图 + "再分析 2 条开启趋势"引导。

### 7.2 多链接输出

- **对比矩阵热力图**：行=维度，列=链接，颜色深浅=分数
- **差异瀑布图**：以「我的」为基线，对标增量显示
- **共性提取卡片**：AI 自动提取「这 N 条的 X 个共同特征」
- **雷达图叠加**：上限 3 条实线，其余虚线
- **趋势折线**：维度时序 + 赛道均值参照线 + 拐点标注

# 第四篇：后端 AI 质量

## 8. 四层架构

| 层 | 做什么 | 快速扫描 | 深度分析 |
|----|--------|---------|---------|
| 1 自检循环 | 生成→自检→修正→输出+置信度 | 跳过 | ✅ |

| 3 价值评分 | 改善杠杆 = 空间×(1-难度)×收益 | ✅ | ✅ |
| 4 推理可视化 | Agent 状态/推理链/证据/验证标识 | 跳过 | ✅ |

### 各模式 LLM 调用量

| 模式 | 调用结构 | 耗时 |
|------|---------|------|
| 快速扫描 | 3 Agent + Chief | 30-60s |
| 深度分析 | 6 Agent + Chief + 自检 + 验证 | 60-180s |
| 对比分析 | (3-6)×N + 对比 Agent + Chief | 90-180s |
| 矩阵分析 | (3-6)×N + 聚类 Agent + Chief | 120-300s |
| 趋势识别 | 轻量×N + 时序 Agent | 120-300s |
| 差距定位 | 深度(我)+轻量(对标)×N+差距 Agent | 90-180s |

## 附录：设计决策速查

| 决策 | 选择 | 原因 |
|------|------|------|
| 快速/深度 | 32px 分段控件 | 策略选择非核心操作 |
| 默认模式 | 智能推荐 | 降低认知负担 |
| 默认深度 | 深度分析 | 类比 Google 不让你选 |
| 默认视图 | 差分报告 | 绝对值无意义 |
| 标准线 | 3 条固定 + 权重动态 | 参照系稳，权重自演进 |
| 标准演进 | 7 因子全自动 | 数据驱动 |
| 因子保底 | 单维不低于基准 20% | 防连乘归零 |
| 冷启动 | 静态模式，逐步解锁 | 越用越聪明 |
| 平台检测 | 排除噪声+全行业趋势 | 防误判 |
| 采纳追踪 | 自动推断 | 零摩擦 |
| 数据源 | 抽象总线+端口实现 | 先跑通现有 |
| 结果复用 | 30天缓存 | 减少重复花费 |
| 免费额度 | 3(Free)/30(PRO)/月 | 功能不阉割 |
| 账号分析 | 预留，另立方案 | 不混入当期 |
