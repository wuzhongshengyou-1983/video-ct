## 5.2 智能化核心：5 大 AI 引擎

### 引擎 1：CT 诊断引擎（无人工干预）

**触发条件：** 用户上传/提交视频
**处理时长：** 平均 90 秒、P95 < 5 分钟
**输出：** 6 维评分报告 + 病灶定位 + 修复建议

**自动化链路：**
```
视频提交 → 排队（Celery 队列）
        → 抽帧 + OCR + ASR（并行，3 个 Worker）
        → 视觉模型分析（异步调用）
        → LLM 合并分析（CT 6 维 prompt）
        → JSON Schema 校验
        → 模板渲染 PDF/H5
        → 推送通知（小程序订阅消息 / 短信 / 邮件）
```

### 引擎 2：对标比对引擎（每日跑批 + 实时按需）

**触发条件：**
- 每日 03:00 自动跑批（更新所有客户的对标差距）
- 用户手动刷新（实时按需）

**处理逻辑：**
```python
# 伪代码
for 客户 in 全部付费客户:
    赛道_top10 = 拉取(客户.赛道)
    用户_data = 拉取(客户.账号, 近30天六大指标)
    对标_data = 计算均值(赛道_top10, 近30天六大指标)
    差距 = (对标_data - 用户_data) / 对标_data
    差距进度 = 差距 - 上月差距
    存入档案(客户.档案_id, 本月差距, 差距进度)
    if 差距进度 > 0 持续两月:
        触发预警(客户.id, 类型='差距扩大')
```

### 引擎 3：智能预警引擎（实时）

**监测指标：** 6 大数据 + 视频更新频次 + 评论情绪

**预警分级：**
| 级别 | 触发条件 | 推送方式 | 谁处理 |
|---|---|---|---|
| 🟢 提示 | 单日数据正常波动 | App 内消息 | 无需 |
| 🟡 注意 | 单条视频数据低于自身均值 30% | 小程序订阅消息 | 用户自查 |
| 🟠 警告 | 连续 3 条视频数据下滑 | App + 短信 | MAX 顾问介入 |
| 🔴 严重 | 账号疑似限流（数据断崖） | App + 短信 + 电话 | MAX 顾问 1h 内介入 |

### 引擎 4：内容生成引擎（钩子/标题/封面/脚本）

**模型路由：**
```
用户档位 → 选模型 → 用 prompt 模板 → JSON 输出 → 后处理 → 返回
PRO     → Qwen-Plus → 库存模板 → 5 个候选 → 用户挑选
MAX     → Qwen-Max → 定制 prompt（注入用户人设档案） → 5 个候选 + 推荐 1 条
加购    → Claude Opus → 全量上下文 → 完整脚本
```

### 引擎 5：报告自动生成引擎

**报告类型与生成方式：**

| 报告 | 长度 | 生成方式 | 输出格式 | PRO | MAX |
|---|---|---|---|---|---|
| 单条 CT 诊断 | 2–3 页 | 模板填空 | H5 + PDF | ✅ | ✅ |
| 月度差距报告 | 4–6 页 | AI 写 + 图表 | PDF | ✅ | ✅ |
| 月度成长档案更新 | 2 页 | 数据填充 | 在线表格 | ⚠ 6 月滚动 | ✅ 终身 |
| 季度战略报告 | 8–12 页 | AI 草稿 + 顾问润色 | PDF + PPT | ❌ | ✅ |
| 年度成长白皮书 | 30+ 页 | 顾问主撰 + 数据图表 | PDF + 印刷版 | ❌ | ✅ 年卡 |

## 5.3 AI + 人工协同的 3 种模式

### 模式 A：纯自动化（PRO 主线）
- AI 端到端，无人工干预
- 适用：标准化诊断、月度复盘、模板生成
- 客服仅在客诉时介入

### 模式 B：AI 草稿 + 人工复审（MAX 标准）
- AI 先出 80% 内容，顾问 30 分钟内审核 + 补充 20%
- 适用：MAX 月度复盘报告、季度战略报告
- 复审失败率目标 < 10%

### 模式 C：AI 辅助 + 人工主导（MAX 高阶）
- AI 仅提供数据看板和资料整理
- 顾问主导分析和决策
- 适用：年度白皮书、矩阵布局、危机处理、头部资源对接

## 5.4 数据反馈闭环：AI 越用越聪明

```
┌───────────────────────────────────────────────────────┐
│  ① AI 出 CT 报告                                       │
│         ↓                                              │
│  ② 用户对报告点评（👍 / 👎 / 评分）                    │
│         ↓                                              │
│  ③ 用户实际执行情况（改了/没改）                       │
│         ↓                                              │
│  ④ 改完之后的数据变化（六大指标变好/变差）             │
│         ↓                                              │
│  ⑤ 顾问审核（仅 MAX）：报告对/不对、建议有效/无效      │
│         ↓                                              │
│  ⑥ 数据回流到训练集 → fine-tune / RAG 更新           │
│         ↓                                              │
│  ⑦ 下个版本 AI 更准、更具个性化                        │
└───────────────────────────────────────────────────────┘
```

**关键指标：** AI 报告满意度（用户 + 顾问双打分）必须从初版 75% 提升到 6 个月后 90%+

## 5.5 自动化基础设施

### 任务编排
- **Celery + Redis Broker**：异步任务（诊断/报告生成）
- **APScheduler**：定时任务（每日跑批 / 预警检测）
- **n8n / Dify**：低代码工作流（运营场景，如客户欢迎流程）

### 监控告警
- **Prometheus + Grafana**：系统指标
- **Sentry**：业务异常
- **PagerDuty / 飞书机器人**：告警通知
- **客户行为埋点**：自研 + Sensors（神策）

### 客户运营自动化
- **企微 SCRM**：客户分层标签 + 自动 SOP 推送
- **客户成长事件流**：触发对应营销动作
  - 注册 30 分钟未付费 → 推送限时优惠
  - 付费 7 天未提交首份视频 → 顾问/客服回访
  - 月度差距连续缩小 3 个月 → 推送升级邀请
  - 续费日前 7 天 → 推送续费提醒 + 优惠
  - 续费日前 3 天 → 顾问主动联系（MAX）

## 5.7 自动化的 3 条铁律

1. **AI 必须可解释**：所有 AI 决策必须给出推理过程，让用户和顾问能 review
2. **AI 必须可纠正**：用户和顾问的反馈必须能在 1 周内进入 prompt/RAG 更新
3. **AI 失败必须降级**：任何 AI 不可用情况下，PRO 自动延期、MAX 顾问兜底
