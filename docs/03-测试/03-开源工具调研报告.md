### 选型依据对照表

| 你的需求 | 用什么满足 | 依据 |
|---|---|---|

## 四、关于「模拟真人」的务实说明

搜索结果表明：**目前没有一个成熟的（>1000 star）、专门做「persona-based / 模拟真人网站测试」的开源工具**。最接近的 AutoUX（<100 star）、ValidateMe（<100 star）、Stella（<50 star）都是个人项目，功能设计很好但不足以用于生产环境。

**务实方案是：借鉴这些项目的 persona 设计理念，自己用 Playwright + Midscene.js 实现。**

具体做法：
1. **Persona 用 JSON 定义**：年龄、技术水平、耐心度、设备、网络、目标意图、行为参数
2. **注入到 Playwright context**：每个 persona 创建独立的 browser context，配置对应的 viewport、userAgent、network、操作延迟
3. **Midscene 自然语言驱动**：不用写死 CSS 选择器，用 `aiAction()` 描述用户意图
4. **并行执行**：Playwright worker 并行跑 8 个 persona

这样既保持了工具的成熟度和可靠性（88k star + 13k star 的组合），又实现了「模拟真人测试」的目标。

## 五、画像配置设计（从成熟工具借鉴的精华）

综合 AutoUX 的 25+ 画像配置、ValidateMe 的 7 维评分体系、Stella 的 7 种预设人格，总结出以下画像参数模型：

```json
{
  "id": "persona-3-creator",
  "name": "创作者",
  "profile": {
    "age": 25,
    "techLevel": "medium-high",       // 影响操作速度
    "occupation": "短视频博主",
    "goal": "上传视频做 CT 诊断"       // 决定测试路径
  },
  "behavior": {
    "patienceMs": 5000,               // 等待超时阈值
    "clickDelayMs": {"min": 400, "max": 1500}, // 模拟真人操作间隔
    "scrollSpeed": "normal",          // 滚动速度
    "abandonRate": 0.15               // 放弃概率
  },
  "device": "iPhone 12",             // Playwright 设备预设
  "network": "4G"                     // Playwright 网络预设
}
```
