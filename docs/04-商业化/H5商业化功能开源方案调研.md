## 调研1 — 短信验证码服务

### 2025年关键背景：工信部新规

四大平台（阿里云/腾讯云/云片/网易云信）在工信部2025年新规后，**纯个人开发者全部无法开通国内短信签名**。要求企业营业执照，且「网站」「公众号」「小程序」「测试」等签名类型被下线。

### 推荐方案：Spug 推送助手 (push.spug.cc)

**理由**：
- GitHub 15k+ Star 开源运维平台 Spug 团队出品，信誉可靠
- 微信扫码即用，**无需营业执照、对公账户、企业认证**
- 0.05元/条，新注册送 50 条免费测试
- REST API，三步接入
- 个人开发者唯一真正可用的国内短信方案

**限制**：
- 签名固定为「Spug通知」，无法自定义
- 单手机号频率限制：1条/分钟、5条/小时
- 仅支持中国大陆手机号

### 备选方案

阿里云「号码认证-短信认证接口」（2025.11 推出）：不需要企业资质和签名审核，但不能自定义签名，产品较新。

### Express 中间件代码示例

```javascript
// sms-auth.js — 短信验证码中间件（基于 Spug 推送助手）
const axios = require('axios');
const crypto = require('crypto');

const SMS_CONFIG = {
  apiUrl: process.env.SPUG_SMS_URL || 'https://push.spug.cc/send/YOUR_TEMPLATE_ID',
  ttl: 300,  // 验证码有效期5分钟
  resendInterval: 60,  // 重发间隔60秒
};

const codeStore = new Map(); // 生产环境换 Redis

// 中间件1: 发送验证码
async function sendSmsCode(req, res) {
  const { phone } = req.body;
  if (!phone || !/^1[3-9]\d{9}$/.test(phone)) {
    return res.status(400).json({ code: 1, message: '手机号格式不正确' });
  }

  const record = codeStore.get(phone);
  if (record && (Date.now() - record.sendAt) / 1000 < SMS_CONFIG.resendInterval) {
    return res.status(429).json({ code: 2, message: '发送过于频繁，请稍后再试' });
  }

  const code = crypto.randomInt(100000, 999999).toString();

  try {
    const { data } = await axios.get(SMS_CONFIG.apiUrl, {
      params: { code, targets: phone },
      timeout: 10000,
    });
    if (data.code !== 200 && data.code !== 0) {
      return res.status(502).json({ code: 3, message: '短信发送失败' });
    }
    codeStore.set(phone, { code, expiresAt: Date.now() + SMS_CONFIG.ttl * 1000, sendAt: Date.now() });
    return res.json({ code: 0, message: '验证码已发送', ttl: SMS_CONFIG.ttl });
  } catch (err) {
    return res.status(502).json({ code: 3, message: '短信服务异常' });
  }
}

// 中间件2: 校验验证码
function verifySmsCode(req, res, next) {
  const { phone, code } = req.body;
  if (!phone || !code) {
    return res.status(400).json({ code: 1, message: '手机号和验证码不能为空' });
  }
  const record = codeStore.get(phone);
  if (!record) return res.status(401).json({ code: 4, message: '请先获取验证码' });
  if (Date.now() > record.expiresAt) {
    codeStore.delete(phone);
    return res.status(401).json({ code: 5, message: '验证码已过期' });
  }
  if (record.code !== String(code)) {
    return res.status(401).json({ code: 6, message: '验证码错误' });
  }
  codeStore.delete(phone);
  req.verifiedPhone = phone;
  next();
}

// 路由使用
// router.post('/api/sms/send', sendSmsCode);
// router.post('/api/auth/login', verifySmsCode, loginHandler);

module.exports = { sendSmsCode, verifySmsCode };
```

## 调研2 — 微信 H5 内打开的最佳实践

### JS-SDK 接入流程

**签名链路**（决定 99% 失败率的环节）：
```
前端传当前页面 URL → 后端用 appId+appSecret 获取 access_token
→ 用 access_token 获取 jsapi_ticket
→ noncestr + jsapi_ticket + timestamp + url 做 SHA1 生成 signature
→ 返回 {appId, timestamp, nonceStr, signature} 给前端
→ wx.config() → wx.ready()
```

**关键约束**：
- `access_token` 有效期 7200s，每日限 2000 次，必须缓存
- `jsapi_ticket` 有效期 7200s，建议缓存 7000s
- 签名 URL 必须去掉 `#` 之后的部分
- iOS 用**第一次进入 webview 的入口 URL**（SPA 路由变化后不变）
- **timestamp 是秒级时间戳，不是毫秒**

### 常见坑

| 问题 | 解决方案 |
|------|---------|
| 微信浏览器顽固缓存 HTML | Nginx `Cache-Control: no-cache` + 入口带版本参数 `?v=20250523` + 前端轮询 `/version.json` |
| iOS 缓存比 Android 更顽固 | 多方案组合，入口改文件名最彻底 |
| `invalid signature` 高频错误 | 排查：URL 去 hash → URL encode → iOS 用入口 URL → NTP 同步服务器时间 → Redis 缓存 ticket |
| 分享图片不显示 | 必须 HTTPS 绝对 URL，200x200+，英文文件名，服务器允许微信 UA |
| 旧分享接口部分安卓不生效 | `jsApiList` 同时加新旧两套接口名 |
| iOS 底部横条遮挡 | `viewport-fit=cover` + `env(safe-area-inset-bottom)` |

### 测试号 — 开发阶段最佳选择

微信公众平台测试号：免费、无需认证、扫码即得。**能覆盖火眼金睛全部核心功能**：
- JS-SDK 全部接口（分享、图片、音频、定位）
- 网页授权 OAuth（获取 openid + 用户信息）
- 自定义菜单、客服消息

**不能做的**：微信支付、超 20 人关注、正式发布

### 微信支付 — 个人开发者的真实门槛

| 支付方式 | 要求 | 个人开发者能做吗 |
|----------|------|:--------------:|
| JSAPI 支付（微信内） | 认证服务号 + 商户平台 | **不能** |

### 最小可用接入代码

**后端 Express 签名服务**：

```javascript
const express = require('express');
const crypto = require('crypto');
const axios = require('axios');

let tokenCache = { access_token: '', expires: 0 };
let ticketCache = { jsapi_ticket: '', expires: 0 };

async function getAccessToken(appid, secret) {
  if (tokenCache.access_token && Date.now() < tokenCache.expires) return tokenCache.access_token;
  const { data } = await axios.get(
    `https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=${appid}&secret=${secret}`
  );
  tokenCache = { access_token: data.access_token, expires: Date.now() + (data.expires_in - 200) * 1000 };
  return data.access_token;
}

async function getJsApiTicket(appid, secret) {
  if (ticketCache.jsapi_ticket && Date.now() < ticketCache.expires) return ticketCache.jsapi_ticket;
  const token = await getAccessToken(appid, secret);
  const { data } = await axios.get(
    `https://api.weixin.qq.com/cgi-bin/ticket/getticket?access_token=${token}&type=jsapi`
  );
  ticketCache = { jsapi_ticket: data.ticket, expires: Date.now() + (data.expires_in - 200) * 1000 };
  return data.ticket;
}

app.get('/api/wx-sign', async (req, res) => {
  const url = decodeURIComponent(req.query.url || '');
  const ticket = await getJsApiTicket(APPID, SECRET);
  const nonceStr = crypto.randomBytes(8).toString('hex');
  const timestamp = Math.floor(Date.now() / 1000);
  const signStr = `jsapi_ticket=${ticket}&noncestr=${nonceStr}&timestamp=${timestamp}&url=${url}`;
  const signature = crypto.createHash('sha1').update(signStr).digest('hex');
  res.json({ appId: APPID, timestamp, nonceStr, signature });
});
```

## 调研3 — 视频数据采集开源方案

### GitHub 开源项目现状

抖音/快手数据采集类 GitHub 仓库普遍面临：
- 平台反爬升级频繁，项目维护困难，多为 **archived/停更** 状态
- 抖音系（含 TikTok）采集项目被封禁下架概率最高
- 幸存项目多为研究性质，稳定性和数据完整性无法保证

### Puppeteer/Playwright 可行性

| 维度 | 评估 |
|------|------|
| 反爬对抗 | 抖音/快手有极强的无头浏览器检测（WebDriver 痕迹、Chrome Runtime 特征、Canvas 指纹） |
| 绕过难度 | 需要持续对抗升级（修改 CDP、伪造浏览器指纹、模拟人类行为），个人开发者难以持续维护 |
| 成功率 | 低且不稳定，IP 封禁风险高 |
| 移动端模拟 | 抖音 Web 版数据不完整，核心功能在 App 端，Web 采集数据量有限 |

### 法律风险评估 ⚠️

| 维度 | 风险定级 |
|------|:------:|
| 爬取公开视频元数据（个人研究） | 中风险 |
| 爬取用户评论/个人信息 | **高风险（PIPL 违法）** |
| 商业用途数据采集 | **高风险** |
| 绕过平台反爬措施 | **高风险（破坏计算机信息系统罪）** |
| 大规模批量采集 | **极高风险（已有 3 年以上判例）** |

**已知判例**：2024-2025 年已有至少 3 起因大规模爬取短视频平台数据被判「非法获取计算机信息系统数据罪」的案件，刑期 3 年以上。

### 推荐方案

**主路径：平台官方 API**

| 平台 | 官方 API | 能获取 | 申请门槛 |
|------|---------|--------|---------|
| 抖音开放平台 | 视频管理 API、数据开放服务 | 自己账号的视频数据、播放量、互动 | 开发者认证 |
| 快手开放平台 | 视频 API、数据 API | 自己账号的视频数据 | 开发者认证 |

**核心逻辑**：火眼金睛是「帮用户分析他们自己的视频」，这正好符合平台官方 API 的授权模式——OAuth 授权后读取用户自己的数据，完全合法合规。

**辅路径：TikHub / 第三方 API 补充**

对于官方 API 不提供的竞品对比数据，可选择性使用第三方 API 做单条查询补充。

### 结论

| 方案 | 可行性 | 法律风险 | 稳定性 |
|------|:-----:|:------:|:-----:|
| 官方 OAuth API | 高 | 无风险 | 高 |
| 第三方 API（TikHub 等）| 中 | 低风险 | 中 |
| GitHub 开源爬虫 | 低 | 中-高风险 | 低 |
| 自建 Puppeteer 方案 | 低 | 高风险 | 极低 |

**推荐路径：官方 OAuth API（主）+ 第三方 API 单条查询（辅）**，这是唯一合法且稳定的方案。自建爬虫在当前司法环境下风险极高。

## 调研4 — 前端诊断报告导出方案

### 方案对比

| 维度 | html2canvas+jsPDF | Puppeteer 服务端 | ECharts getDataURL+拼装 |
|------|:---:|:---:|:---:|
| 图表保真度 | 中（Canvas 截图有失真） | 高（完整渲染） | 高（原生导出） |
| 移动端 H5 性能 | 差（内存占用大） | N/A（服务端） | 好 |
| 微信浏览器兼容 | 差（html2canvas 已3年未更新，971个 open issues） | 无关 | 好 |
| 服务器成本 | 无 | 高（需 Headless Chrome） | 低（仅 OSS 存储） |
| 开发复杂度 | 中 | 高 | 中 |
| 文件大小 | 大 | 中 | 可控 |
| 多图表报告 | 支持 | 支持 | 需手动拼接 |

### 推荐方案：ECharts getDataURL + Canvas 拼装 + OSS 上传

**理由**：
1. ECharts 的 `getDataURL()` 原生导出图表为高清 PNG，保真度最高
2. 用 Canvas API 绘制文字段落和数据区域，将多个图表拼接为完整报告
3. Canvas 导出 PNG → 上传 OSS 获取真实 URL → 微信分享卡片使用 OSS URL
4. html2canvas 已停更 3 年，971 个未解决 Issue，微信浏览器兼容性无保障
5. 纯前端方案，零服务器成本

### 核心代码示例

```javascript
// report-exporter.js — 诊断报告导出（ECharts + Canvas 拼装 + OSS 上传）

async function exportReport(charts, metadata, canvasWidth = 750) {
  // 1. 导出各图表为图片
  const chartImages = await Promise.all(
    charts.map(chart => {
      // ECharts 实例直接导出
      return new Promise((resolve) => {
        const dataURL = chart.getDataURL({ type: 'png', pixelRatio: 2, backgroundColor: '#fff' });
        const img = new Image();
        img.onload = () => resolve(img);
        img.src = dataURL;
      });
    })
  );

  // 2. 计算总画布高度
  let totalHeight = 0;
  const sections = [
    { type: 'title', height: 120 },
    ...chartImages.map((img, i) => ({ type: 'chart', img, height: img.height + 40, index: i })),
    { type: 'footer', height: 80 },
  ];
  totalHeight = sections.reduce((sum, s) => sum + s.height, 0);

  // 3. Canvas 拼装
  const canvas = document.createElement('canvas');
  canvas.width = canvasWidth;
  canvas.height = totalHeight;
  const ctx = canvas.getContext('2d');

  // 背景
  ctx.fillStyle = '#ffffff';
  ctx.fillRect(0, 0, canvasWidth, totalHeight);

  let yOffset = 0;
  for (const section of sections) {
    if (section.type === 'title') {
      ctx.fillStyle = '#1a1a2e';
      ctx.fillRect(0, yOffset, canvasWidth, section.height);
      ctx.fillStyle = '#ffd700';
      ctx.font = 'bold 36px "PingFang SC", sans-serif';
      ctx.textAlign = 'center';
      ctx.fillText('火眼金睛 · 视频诊断报告', canvasWidth / 2, yOffset + 60);
      ctx.fillStyle = '#ffffff';
      ctx.font = '20px "PingFang SC", sans-serif';
      ctx.fillText(`诊断时间: ${metadata.date}`, canvasWidth / 2, yOffset + 95);
      yOffset += section.height;
    } else if (section.type === 'chart') {
      ctx.drawImage(section.img, 0, yOffset, canvasWidth, section.img.height);
      yOffset += section.height;
    } else if (section.type === 'footer') {
      ctx.fillStyle = '#f5f5f5';
      ctx.fillRect(0, yOffset, canvasWidth, section.height);
      ctx.fillStyle = '#999';
      ctx.font = '16px "PingFang SC", sans-serif';
      ctx.textAlign = 'center';
      ctx.fillText('火眼金睛 AI 诊断系统 · 如意孙小圣', canvasWidth / 2, yOffset + 40);
    }
  }

  // 4. 导出为 Blob 并上传
  return new Promise((resolve) => {
    canvas.toBlob(async (blob) => {
      const formData = new FormData();
      formData.append('file', blob, `report_${Date.now()}.png`);

      // 上传到自己的 OSS 代理接口
      const resp = await fetch('/api/oss/upload', { method: 'POST', body: formData });
      const { url } = await resp.json();
      resolve({ url, blob }); // url 可用于微信分享
    }, 'image/png', 0.95);
  });
}
```
