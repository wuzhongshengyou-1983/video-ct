2. Responsively App 使用规范

open -a ResponsivelyApp         # 启动

2.2 标准工作模式

1
VSCode 启动开发服务器
在 VSCode 集成终端运行 npm run dev，记录端口（通常 3000 / 5173 / 8080）

打开 Responsively，输入本地地址

配置常用设备组

3. Eruda 使用规范

3.1 安装

# 方式 A：npm（推荐，构建工具可 tree-shake）
npm install eruda --save-dev

# 方式 B：CDN（纯 HTML 项目）
<script src="https://cdn.jsdelivr.net/npm/eruda@3/eruda.min.js"></script>

3.2 标准接入代码（React / Vue 项目）

// src/main.js 或 src/index.js 顶部
if (import.meta.env.DEV) {         // Vite 写法
  import('eruda').then(({ default: eruda }) => eruda.init())
}

// Create React App 写法
if (process.env.NODE_ENV === 'development') {
  require('eruda').init()
}

3.3 标准接入代码（纯 HTML 项目）

<!-- 放在 </body> 前，生产上线时删除整个 script 块 -->
<script>
  var isDev = window.location.hostname === 'localhost'
            || window.location.hostname === '127.0.0.1'
            || window.location.search.includes('debug=true')

  if (isDev) {
    var s = document.createElement('script')
    s.src = 'https://cdn.jsdelivr.net/npm/eruda@3/eruda.min.js'
    s.onload = function() { eruda.init() }
    document.head.appendChild(s)
  }
</script>
💡 高级技巧：?debug=true 生产临时调试 上面代码中加入了 window.location.search.includes('debug=true')。 线上遇到 bug 时，URL 后加 ?debug=true 即可临时唤出调试面板，不需要重新发布。 注意：仅在内部测试时用，不要公开此技巧。

4. 安全规范（商业项目强制）

🔴 P0 红线：Eruda 严禁进入生产环境 Eruda 面板暴露在生产环境会让任意用户看到：完整 API 请求体（含用户 Token）、LocalStorage 内的敏感数据、完整 DOM 结构。 这是一个严重的信息泄露漏洞，任何情况下不允许带 eruda.init() 上生产。

4.1 构建验证（CI 必加）

# package.json scripts 中加入构建前检查
"prebuild": "grep -r 'eruda.init' src/ && echo '❌ 发现 eruda.init，禁止构建' && exit 1 || echo '✅ 无 eruda.init'"

4.2 .gitignore 规范

# .gitignore 已有，确认包含：
node_modules/
.env.local
.env.production
dist/
build/

4.3 BrowserSync 网络安全（如使用）

# 限制只在本机访问，防止局域网其他设备看到你的代码
npx browser-sync start --server --host localhost

# 如需局域网真机测试，用完立即关闭
npx browser-sync start --server  # 用完 Ctrl+C

6. 上线前检查清单

  • 代码中无 eruda.init()（或已被 NODE_ENV 条件包裹）
  • eruda 在 package.json 中是 devDependencies，不是 dependencies
  • 生产 build 产物中搜索不到 "eruda" 字符串
  • BrowserSync 未在生产服务器上运行
  • 已在 iPhone SE（375px）最小屏下测试布局无破坏
  • 已在 Android（360px）下测试触摸交互正常
  • Responsively App 截图留存，作为视觉验收记录
  • 不能将 ?debug=true 链接分享给终端用户
  • 不能在生产环境 URL 中包含 debug=true

7. 常见问题

Q：Responsively 打开页面是空白的？

检查开发服务器是否正在运行（npm run dev），以及端口是否正确。Responsively 只是一个浏览器，需要你的服务器先跑起来。

Q：Eruda 面板不出现？

1. 检查 CDN 是否可达（网络问题）。2. 检查 eruda.init() 是否在 DOM 加载后执行。3. 检查是否被 Content Security Policy 拦截（控制台会有报错）。

Q：手机访问 localhost 打不开？

localhost 只能本机访问。手机测试需要用 BrowserSync 或查本机局域网 IP（ifconfig | grep inet），用 http://192.168.x.x:3000 访问。

Q：Responsively 和 Chrome DevTools 的区别？

Chrome DevTools 的设备模拟是单设备切换，一次只看一个。Responsively 同时显示所有设备，修改 CSS 时能一眼看到所有尺寸的变化，效率更高。
