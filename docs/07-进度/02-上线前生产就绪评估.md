### 4.3 video-ct 自定义规则

项目中创建 `rules/custom-video-ct.yaml`：

```yaml
# video-ct 自定义规则 — AI 服务 / 视频 pipeline / 多端部署

rules:
  - id: ai-llm-fallback
    severity: high
    category: reliability
    description: "AI/LLM 服务必须有降级策略"
    signals: [llm_fallback_configured, llm_retry_with_backoff]
    why: "SiliconFlow / DeepSeek API 不可用时，分析任务不能全部失败"

  - id: video-pipeline-retry
    severity: high
    category: reliability
    description: "视频抓取/OCR/ASR pipeline 需要重试和死信队列"
    signals: [celery_retry_configured, dead_letter_queue_exists]
    why: "外部视频平台抓取可能超时/限流"

  - id: websocket-reconnect
    severity: medium
    category: resilience
    description: "WebSocket 推送需要客户端重连 + 服务端心跳"
    signals: [ws_heartbeat_configured, ws_client_reconnect_logic]
    why: "移动端网络切换导致断连，分析结果推送不能丢失"

  - id: multi-tenant-isolation
    severity: medium
    category: security
    description: "C端/顾问端/运营端数据隔离"
    signals: [tenant_scoping_in_queries, rbac_tested]
    why: "顾问不能访问其他顾问的客户数据"

  - id: mobile-h5-resilience
    severity: medium
    category: ux
    description: "H5 弱网适配 + 离线兜底"
    signals: [service_worker_registered, api_retry_on_network_error, skeleton_loading_states]
    why: "移动端用户可能在弱网环境使用视频上传功能"

  - id: ai-response-validation
    severity: medium
    category: safety
    description: "AI 返回结果需要校验层"
    signals: [ai_response_schema_validation, ai_content_filter]
    why: "AI 返回格式错误或敏感内容时不能直接展示给用户"
```

### 4.4 CI/CD 集成

在 `.github/workflows/production-readiness.yml`：

```yaml
name: Production Readiness Scan
on:
  pull_request:
    branches: [main]
  push:
    tags: ['v*']

jobs:
  readiness:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.22'
      - name: Install pr scanner
        run: |
          git clone https://github.com/chuanjin/production-readiness /tmp/pr
          cd /tmp/pr && go build -o pr ./cmd/pr
          sudo mv pr /usr/local/bin/
      - name: Run readiness scan
        run: pr scan . > readiness-report.md
      - name: Upload report
        uses: actions/upload-artifact@v4
        with:
          name: readiness-report
          path: readiness-report.md
```
