## 二、智能分析层——诊断不只打分

### 2.1 单条视频诊断（v2.5 已有）

```
输入 → 6维评分 + 秒级病灶 + 优先级排序 + 赛道差距
```

### 2.2 跨视频模式识别（v2.5→v3 新增）

**核心思路**：不是每次都从零诊断，而是把每条视频放进用户的历史坐标系里看。

```
本次诊断 vs 历史平均 vs 历史最佳 vs 赛道均值
```

**四个维度**：

#### A. 分数趋势
```
最近5条的 overall_score：
72 → 68 → 75 → 70 → 78（本次）
结论：「上升趋势，比你的均分 72.6 高 5.4 分，接近你历史最佳 80 分」
```

#### B. 维度变化
```
内容内核: 58→65→72→68→75（本次）「稳定上升，你越来越会讲内容了」
表层观感: 70→68→62→60→58（本次）「连续4条下降，封面问题是你的慢性病」
```

#### C. 复发问题检测
```
「开头钩子弱」这个病灶在最近5条视频中出现了 4 次
→ 不是偶然失误，是习惯性短板
→ 建议：系统性学习钩子公式，不要只修这一条视频
```

#### D. 赛道定位偏移
```
你的「人设话术」维度持续低于赛道均值
→ 可能你的人设标签不够清晰
→ 对比你的历史最佳那条（标题「XXX」），那条的人设话术 78 分
```

### 2.3 诊断结论分层

不再是一份平铺的报告，而是三层结论：

```
第一层（30秒看完）：总分 + 一句话结论 + 最优先改什么
第二层（3分钟看完）：6维每项的「变化趋势 + 关键问题 + 对比」
第三层（深度看）：逐秒病灶 + 修复方案 + 历史对比数据
```

## 四、历史结合方案

### 4.1 数据存储（v2.5 DB 已有，需扩展）

```sql
-- 每份报告的汇总趋势快照，供快速对比
reports 表（已有）:
  overall_score, grade, dimensions (JSON), findings (JSON),
  content_ai (JSON), benchmark_context (JSON),  -- v2.5 新增
  data_confidence, cost_yuan                     -- v2.5 新增

-- 需要新增：用户级趋势缓存（避免每次重算）
user_trends 表（v3新增）:
  user_id, last_5_scores (JSON), last_5_grades,
  chronic_issues (JSON),     -- 复发≥3次的问题
  streak_dimensions (JSON),  -- 连续上升/下降的维度
  calculated_at
```

### 4.2 历史对比计算

```
每次新诊断完成后，异步计算：

1. 取用户最近 5 份报告
2. 计算每条维度分数的趋势（线性回归方向）
3. 对比本次 vs 均分 / 最佳 / 上次
4. 检测病灶是否在历史中重复出现（同一维度+相似问题描述）
5. 生成对比结论文本
6. 写入 user_trends 缓存
```

### 4.3 历史洞察输出

```
「你已诊断 12 条视频」
  最好成绩: 82 分（3月15日那条「如何选品」）
  最弱维度: 变现预埋（12条平均 48 分，赛道均值 55）
  进步最快: 人设话术（从 52→70，+18）
  慢性病: 开头钩子弱（出现 8/12 次），封面文字小（出现 6/12 次）
  建议: 下个阶段你不需要再优化内容内核了（已达赛道前30%），
        集中精力攻克变现预埋，这是你从 L3→L4 的关键卡点
```

## 六、v3 兼容性设计

**核心原则**：v2.5 的实现就是 v3 的子集，不做推倒重来。

| v2.5 | v3 |
|------|-----|
| 三源并行（TikHub+WebFetch+LLM） | 加 F6 QoE 变成四源，Registry 不变 |
| `content_ai` / `benchmark_context` JSON 列 | v3 直接读写同列，无需迁移 |
| `_pipeline.history_compare` 字段 | v3 升级为 `user_trends` 表，读取来源不变 |
| CT prompt 注入历史数据 | v3 prompt 增加完播率等新字段 |
| `GET /me/growth` 基础版 | v3 升级为完整成长报告 |
| SourceTransparency 六族面板 | v3 增加 F6 行，结构不变 |

**v2.5 的数据结构就是 v3 的数据结构。v3 只做增量：F6、user_trends 表、阶段报告。**
