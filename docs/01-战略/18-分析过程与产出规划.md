## 一、全链路分析总览

```
用户粘贴视频链接
        │
        ▼
POST /api/v1/diagnoses/submit
        │ status: queued
        ▼
┌───────────────────────────────────────────────────────────┐
│                   diagnose_video()                        │
│                                                           │
│  STEP 1  F1 平台采集                    ≤ 15s 熔断        │
│  │       MultiSourceFetcher                               │
│  │       TikHub API + (官方API·待接)                      │
│  │       → VideoMeta  confidence=high/medium/low          │
│  │                                                        │
│  ▼ (并行)                                                 │
│  STEP 2a F3 AI内容理解                  ≤ 15s 熔断        │
│  │       Qwen VL 封面分析 + SenseVoice ASR                │
│  │       → ContentAnalysis                                │
│  │                                                        │
│  STEP 2b F4 赛道基准                    ≤ 10s 熔断        │
│  │       LLM 知识基准（内存缓存/天）                        │
│  │       → BenchmarkContext                               │
│  │                                                        │
│  ▼ (合并)                                                 │
│  STEP 3  F5 CT 诊断                     ≤ 25s 熔断        │
│  │       DeepSeek CT Prompt                              │
│  │       ├── 视频基础数据（F1）                            │
│  │       ├── 互动率计算（点赞/评论/分享/收藏）              │
│  │       ├── 封面画面 + 转写文本（F3）                     │
│  │       ├── 钩子策略 + 语速（F3）                        │
│  │       └── 赛道均值基准（F4）                           │
│  │       → CTReport  6维 + 病灶 + 优先级 + 差距            │
│  │                                                        │
│  STEP 4  成本聚合 + 数据透明度组装                         │
│          → FinalReport                                    │
└───────────────────────────────────────────────────────────┘
        │ status: done
        ▼
GET /api/v1/diagnoses/{id}/report  →  ReportOut
```

## 二、各阶段产出定义

### STEP 1 · F1 平台数据采集

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `platform` | str | URL 推断 | douyin/kuaishou/bilibili |
| `title` | str | API | 视频标题 |
| `description` | str | API | 文案描述 |
| `duration_sec` | int | API | 时长（秒） |
| `cover_url` | str | API | 封面图 URL |
| `video_download_url` | str | API | 视频流地址（供 ASR 使用） |
| `play_count` | int | API 多源均值/最大值 | 播放量 |
| `like_count` | int | API 多源 | 点赞数 |
| `comment_count` | int | API 多源 | 评论数 |
| `share_count` | int | API 多源 | 分享数 |
| `collect_count` | int | API 多源 | 收藏数 |
| `creator_nickname` | str | API | 作者昵称 |
| `creator_follower_count` | int | API | 粉丝数 |
| `confidence` | str | MultiSourceFetcher | high/medium/low |
| `source_count` | int | MultiSourceFetcher | 成功响应源数量 |
| `tags` | list[str] | API | 视频标签 |

**降级行为**
- 单源响应 → confidence=medium
- 两源 play_count 差 >20% → confidence=medium
- 两源 play_count 差 >50% → confidence=low + 人工复核队列
- 全部失败 → `AllSourcesUnavailableError`（诊断中止）

---

### STEP 1.5 · OCR（封面文字提取）

| 字段 | 类型 | 模型 | 说明 |
|------|------|------|------|
| `ocr_text` | str | SiliconFlow Vision | 封面所有文字，从上到下 |

**费用估算**：单次 ≈ ¥0.002（图片 token 约 500）

---

### STEP 2a · F3 AI 内容理解

| 字段 | 类型 | 模型 | 说明 |
|------|------|------|------|
| `cover_analysis` | str | Qwen3-VL-32B | 封面画面描述（主体/文字/质量/氛围） |
| `transcript` | str | SenseVoice | 完整语音转写文本 |
| `segments` | list | SenseVoice | 带时间戳的语音分段 |
| `speech_rate_wps` | float | 计算 | 语速（字/秒） |
| `summary` | str | Qwen2.5-72B | 内容核心摘要（2句话） |
| `hook_hint` | str | Qwen2.5-72B | 开头3秒钩子策略（1句话） |
| `content_source` | str | — | qwen_vl+asr / asr_only / vl_only / timeout |

**费用估算**：封面VL ≈ ¥0.004 + ASR ≈ ¥0.008/min + 摘要LLM ≈ ¥0.006 = **¥0.018**

**熔断：整体 ≤ 15s，超时返回 content_source=timeout**

---

### STEP 2b · F4 赛道基准

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `track` | str | 用户输入 | 赛道名称 |
| `platform` | str | F1 | 平台 |
| `benchmarks.like_rate_pct` | float | LLM知识 | 赛道典型点赞率（%） |
| `benchmarks.comment_rate_pct` | float | LLM知识 | 评论率（%） |
| `benchmarks.share_rate_pct` | float | LLM知识 | 分享率（%） |
| `benchmarks.collect_rate_pct` | float | LLM知识 | 收藏率（%） |
| `benchmarks.avg_play_count` | int | LLM知识 | 典型播放量 |
| `benchmarks.avg_duration_sec` | int | LLM知识 | 典型时长 |
| `benchmarks.top10_like_rate_pct` | float | LLM知识 | TOP10 点赞率 |
| `track_characteristics` | str | LLM知识 | 赛道核心特点（2句话） |
| `common_issues` | list[str] | LLM知识 | 该赛道常见问题（3条） |
| `sample_note` | str | — | 数据说明（LLM估算/超时不可用） |

**缓存**：同赛道同平台内存缓存，进程生命周期内复用（v3 改为按天 Redis 缓存）

**费用估算**：LLM ≈ 500 in + 200 out tokens → **¥0.003**（缓存命中则 ¥0）

### STEP 3 · F5 CT 诊断

**输入给 DeepSeek 的 prompt 结构：**

```
【基本信息】标题/时长/作者/粉丝/赛道
【真实数据】播放量 + 四率（点赞/评论/分享/收藏 %）+ 置信度注记
【封面文字 OCR】（F1.5产出）
【AI 内容理解】cover_analysis + transcript + summary + 语速 + hook_hint（F3产出）
【赛道基准】like_rate/comment_rate/.../track_characteristics/common_issues（F4产出）
【用户人设档案】（Persona + RAG知识，可选）
```

**产出 · CTReport：**

## 三、完整报告产出结构（面向前端）

### PRO 报告（¥99/月，每月4次）

```
ReportOut
├── 核心结论
│   ├── overall_score      0-100
│   ├── grade              L1-L6
│   └── summary            医患式总评（200字）
│
├── 视频快照
│   ├── title / duration_sec / platform
│   ├── author (nickname, follower_count)
│   ├── stats (play/like/comment/share/collect)
│   └── data_confidence    置信度标注（high→不展示，medium/low→展示说明）
│
├── 六维评分
│   └── dimensions[6]  score + findings + suggestions
│
├── 时间轴病灶
│   └── lesions[n]  timestamp + dimension + problem + fix
│
├── 修复优先级
│   └── priorities[3]  rank + what_to_do（P0最急）
│
├── 赛道对标
│   └── benchmark_gap  四率差距 + 整体差距%
│
├── AI 内容洞察（F3）         ← v2.5 新增，PRO 摘要版
│   ├── hook_hint          开头钩子策略
│   ├── speech_rate_wps    语速
│   ├── content_summary    内容摘要
│   └── cover_analysis     封面画面描述
│
└── 数据透明度标注
    ├── content_source     qwen_vl+asr / asr_only / timeout
    ├── confidence         数据置信度
    └── cost_yuan          本次诊断费用（运营视角，不对用户展示）
```

### MAX 报告（¥499/月，不限次）

在 PRO 基础上新增：

```
├── 完整 ASR 转写文本        transcript（全文）
├── 赛道深度基准            track_characteristics + common_issues
├── 专属 TOP10 对标         similar_hits[10]（v3 实时刷新）
├── 改稿清单               action_plan（逐条可勾选）
├── 视频手术台              surgery_plan（逐秒对比方案）
└── 顾问批注区              consultant_notes（1v1顾问专属）
```
