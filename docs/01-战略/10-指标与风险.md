## 10.1 北极星指标（North Star Metric）

> **付费用户的"赛道差距缩小百分比"**

```
Δ差距_全公司 = AVG(每位付费客户的月度差距缩小%)

良性区间：每月 -3% ~ -8%
预警阈值：连续 2 月 ≥ 0 → 触发产品复盘
止损阈值：连续 3 月 ≥ 0 → 触发产品重构
```

**为什么这个指标是北极星？**
- 它直接驱动续费率、口碑、ARPU、矩阵扩张
- 它要求公司同时把"AI 准确性""人工服务质量""头部对标新鲜度"做好
- 它无法靠营销技巧造假——必须靠真实交付

## 10.2 指标体系（4 层）

### 层 1：北极星（1 个）
- Δ差距_全公司（每月一次）

### 层 2：一级指标（10 个，必须每周看）

| 指标 | 目标值 | 监控频次 |
|---|---|---|
| 新增付费用户（PRO） | M3: 30/天 | 每日 |
| 新增付费用户（MAX） | M3: 1/天 | 每日 |
| 月续费率（PRO） | ≥ 70% | 每月 |
| 月续费率（MAX） | ≥ 85% | 每月 |
| 月活/付费用户比 | PRO ≥ 70%；MAX ≥ 90% | 每周 |
| AI 报告满意度 | ≥ 85% | 每周 |
| MAX 顾问 NPS | ≥ 60 | 每月 |
| 综合获客 CAC | ≤ 264 元 | 每月 |
| LTV/CAC | ≥ 5 | 每月 |
| 月营收 | 按路线图 | 每周 |

### 层 3：二级指标（30+ 个，按业务模块）

#### 产品健康度
- 单条诊断 P95 延迟 < 5 分钟
- 报告 H5 完成率 > 80%
- 模板下载率
- 客服工单首响时长
- 客服工单解决率

#### 客户成功
- 首次价值时长 TTFV
- 30 天活跃度
- 90 天活跃度
- 月度差距进度均值

#### 营销/增长
- 各渠道 ROI（抖音/视频号/小红书/公众号/转介绍）
- 内容素材 CTR
- 留资到付费转化率
- 升级转化率（PRO → MAX）

#### 财务
- 月毛利率
- 月运营成本
- 现金流跑道（剩余可运营月数）
- 单次客单价

### 层 4：业务过程指标（按部门细化）
- 顾问：人均客户数、人均月度复盘场次、单客平均沟通时长
- AI 团队：模型成本占比、Prompt 迭代次数、Few-shot 库规模
- 内容：周产出量、爆款率
- 法务：合规事件数、应对时长

## 10.4 风险矩阵

### 风险评估方法
- **影响度**（1-5）：1=轻微 / 5=致命
- **概率**（1-5）：1=极低 / 5=极高
- **优先级** = 影响度 × 概率
- 任何 ≥ 12 的风险必须有书面预案

### 风险清单

#### 🔴 致命级风险（优先级 ≥ 15）

| 风险 | 影响 | 概率 | 预案 |
|---|---|---|---|
| **平台政策变化（禁止数据采集）** | 5 | 4 | 多平台分散 + 第三方采购 + 用户授权数据补充 |
| **AI 报告大规模错误引发客诉** | 5 | 3 | 三道质检 + MAX 人工复审 + 满意度兜底赔付 |
| **现金流断裂** | 5 | 3 | M3 启动天使轮、M9 启动 A 轮、储备 6 月跑道 |
| **核心顾问集体离职** | 5 | 3 | 标准化 SOP + 顾问梯队（L1-L4）+ 期权绑定 |

#### 🟠 严重级风险（优先级 9-14）

| 风险 | 影响 | 概率 | 预案 |
|---|---|---|---|
| **竞品快速复刻 6 维 CT 模型** | 4 | 3 | 加速对标库积累 + 案例库 + 顾问 know-how 沉淀 |
| **投放 ROI 长期 < 1** | 4 | 3 | 设单月止损线、多渠道 A/B 测试、强化内容自然流量 |
| **MAX 续费率不达标（< 70%）** | 4 | 3 | 顾问 KPI 与续费率挂钩、流失访谈、产品升级 |
| **个人信息泄露事故** | 5 | 2 | 等保认证、定期渗透测试、应急响应 SOP |
| **被恶意爬虫攻击/DDoS** | 3 | 4 | 阿里云 WAF + CDN 防护、限流 |
| **关键 AI 模型供应商断供** | 4 | 3 | 多模型路由（DeepSeek + Qwen + Claude） |
| **核心创始人/CEO 突发离开** | 5 | 2 | 二级管理梯队、知识沉淀、股权架构合理 |

#### 🟡 一般级风险（优先级 4-8）

| 风险 | 影响 | 概率 | 预案 |
|---|---|---|---|
| 工单系统积压 | 3 | 3 | 自动分配 + 工单优先级 + 排班 |
| 模板版权纠纷 | 3 | 2 | 全部自研 / 正版采购 |
| 退款率超 5% | 2 | 3 | 优化产品体验、限制退款条款 |
| 内容生产瓶颈 | 3 | 3 | 内容外包 + AI 辅助 |
| 顾问培训周期过长 | 3 | 3 | 标准化培训手册 + 视频教程 |

#### 🟢 低风险（监控但不投入资源）
- 服务器扩容跟不上
- 某场直播观看量低于预期
- 个别员工绩效低

## 10.5 应对预案 SOP

### 预案 1：平台政策变化

```
T+0  事件发生（如平台公告禁止爬虫）
T+1  法务 + CEO 评估影响范围
T+3  紧急切换备用数据源（第三方采购数据 + 用户授权数据）
T+7  对外公告：服务方式微调，承诺零中断
T+14 全部受影响客户主动安抚 + 给予赔偿（延期/赠送）
T+30 重构数据采集 SOP，恢复正常
```

### 预案 2：AI 大规模报告错误

```
T+0  客户投诉/内部发现（任一）
T+1h 暂停所有 AI 报告生成
T+3h 技术团队定位问题（Prompt / 模型 / 数据）
T+6h 修复 + 回归测试
T+12h 恢复生产
T+24h 受影响客户主动联系 + 重新生成报告 + 服务期延长 7 天
T+72h 全员复盘 + 长期预防机制（增加质检、降级方案）
```

### 预案 3：现金流告急

```
跑道剩余 9 月 → 启动融资（A 轮）
跑道剩余 6 月 → 砍掉非核心成本（裁员/减少投放）
跑道剩余 3 月 → 启动桥接贷款 / 老股东追加
跑道剩余 1 月 → 启动战略转型（出售业务 / 卖给同行）
```

### 预案 4：核心顾问离职

```
T+0  顾问提出离职
T+1  客户名单冻结
T+3  分配给替补顾问，安排交接会议
T+7  客户 1v1 安抚 + 介绍新顾问
T+30 续费率监控（如下降 > 20%，启动 CEO 关怀）
```

## 10.6 季度复盘机制

### 复盘节奏
- **每周**：增长/产品/顾问三大例会，2 小时
- **每月**：CEO 全员月度复盘，2 小时
- **每季**：股东 + 顾问委员会季度战略复盘，半天
- **每年**：全员年终复盘 + 战略调整，2 天

### 复盘四问
1. **预期 vs 实际**：哪些超预期、哪些不及预期？
2. **根因分析**：为什么超 / 为什么不及？
3. **下一阶段调整**：要保留什么、改变什么、新增什么、删除什么？
4. **机制化**：哪些教训要变成 SOP / 文化 / 红线？

## 10.7 终极风险：被自己打败

| 警示信号 | 应对 |
|---|---|
| 增长压力下开始模糊承诺 | CEO 红线检查、所有话术法务过审 |
| 顾问开始接管账号操作 | 强制培训、KPI 不挂账号涨粉 |
| AI 报告开始走"准确性下降换速度" | 满意度低于阈值立即回退 |
| 为了续费开始推销加购 | 顾问 KPI 与续费挂钩，不与加购挂钩 |
| 创始人开始相信"我们已经成功" | 每季外部对标 + 客户访谈 |

## 10.8 战略级红线（CEO 个人承诺）

不论商业压力多大，永远不做：

1. ❌ 不刷量、不刷粉、不控评
2. ❌ 不承诺"必涨粉、必爆款、必变现"
3. ❌ 不抄袭头部、不搬运
4. ❌ 不收现金、不规避税务
5. ❌ 不剥削顾问/员工
6. ❌ 不做未经审计的数据宣传
7. ❌ 不参与传销式分销
8. ❌ 不背叛用户隐私

> **公司可以慢，但不能歪。**
