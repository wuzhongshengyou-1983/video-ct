## 12.1 战略前提：AI 不是工具，是生产线

### 行业普遍误区

| 同行做法 | 真相 |
|---|---|
| AI = 调一下 GPT 写文案 | 那叫"用 AI"不叫"AI 系统" |
| AI 客服 = 关键词匹配回复 | 那叫"自动回复"不叫"AI 客服" |
| AI 越多越好 | 错。AI 错位调用会拉低质量、暴露成本 |
| AI 一次部署一劳永逸 | 错。AI 必须每周迭代、每月喂养、每季进化 |

### 视频 CT 的 AI 战略原则（4 条）

1. **AI 是核心生产力，不是辅助工具**——80% 工作量必须由 AI 承担，人工只做 AI 干不了的 20%
2. **AI 必须形成飞轮**——客户数据反哺模型，模型反哺产品，产品反哺客户体验
3. **AI 必须可解释、可纠正、可降级**——禁止黑盒 AI、禁止不可推翻的 AI 决策
4. **AI 必须算账**——每个 AI 调用都有成本/收益账，单 Agent 月度 ROI 必须 > 3

### 一句话定位

> **视频 CT 不是一家"用了 AI 的咨询公司"，是一家"AI 公司 + 顾问陪跑"——AI 是引擎，顾问是方向盘。**

## 12.3 8 大 Agent 矩阵（每个机器人都是一个"专员"）

每个 Agent 都遵循 **「LLM + 工具集 + 双层记忆 + 自反思」** 标准架构。

### Agent 1：CT 诊断官（CTRadiologist）
**职责：** 接到一条视频，出 6 维 CT 报告
```yaml
身份: 资深视频影像诊断专家
LLM: pick_model(tier, "chat") → 硅基流动（见 12.2.1 路由矩阵）
工具:
  - extract_frames(video_url)
  - run_ocr(image_list)
  - run_asr(video_url)
  - vision_analyze(image_list)
  - fetch_user_archive(user_id)
  - render_report(json) → PDF/H5
短期记忆: 本次诊断上下文（视频元数据、用户画像）
长期记忆: 该用户历史 30 条诊断、共性病灶
反思机制: 输出前自检 schema 合规、分数合理性、建议可执行性
KPI:
  - 报告生成 P95 < 5 分钟
  - 用户满意度 ≥ 85%
  - 顾问复审通过率 ≥ 90%
```

### Agent 2：对标分析师（BenchmarkAnalyst）
**职责：** 维护 TOP10 对标池，每日跑批六大指标差距
```yaml

工具:
  - crawl_public_data(account_id)

  - detect_trend(time_series)

长期记忆: 各赛道头部数据时间序列
反思机制: 异常值检测（数据是否真实/平台是否反爬）
KPI:
  - 跑批成功率 ≥ 99%
  - 数据准确率 ≥ 95%
```

### Agent 3：人设观察员（PersonaScout）
**职责：** 持续观察用户视频集 + 评论池，输出 IPP 评分与漂移预警

```yaml
长期记忆: 用户人设演进档案（季度快照）
反思机制: 检测"自我标签 vs 粉丝感知"差距，差距 > 30% 触发预警
KPI:
  - 人设漂移识别准确率 ≥ 80%
  - 12 原型匹配准确率 ≥ 85%
```

## 12.4 4 大数据飞轮（越用越聪明的核心）

### 飞轮 1：诊断反馈飞轮（提升 AI 报告质量）

```
① AI 出 CT 报告
   ↓
② 用户对报告打分（👍👎 + 评论）
   ↓
③ 顾问复审打分（MAX 强制）
   ↓
④ 双层反馈合并 → 入"诊断质量评估池"
   ↓
⑤ 每周一 Prompt 工程师审核低分案例
   ↓
⑥ 更新 Prompt 模板 / Few-shot 库 / 输出 Schema
   ↓
⑦ 下版本部署，灰度对照（A/B 各 50%）
   ↓
⑧ 满意度提升 → 全量发布
```
**目标：报告满意度从初版 75% → 6 月内 90%+**

### 飞轮 2：改稿验证飞轮（验证 AI 建议真有效）

```
① AI 给出修改建议（清单 + 优先级）
   ↓
② 用户实际执行（采纳/部分/未执行）
   ↓
③ 改后视频数据回流（六大指标变化）
   ↓
④ 关联到原建议条目 → "建议有效性数据库"
   ↓
```

```
   ↓
⑦ 客户成功 → 公司案例库 → 反哺销售 → 拉新客户
```
**目标：每月行业趋势库新增 ≥ 50 条**

| 项目 | 内容 |

| 责任人 | AI 工程师团队 |
| 战略意义 | **核心壁垒之一**：自有微调模型，对手买不到 |

## 12.6 AI 工作循环：Plan → Tool → Observe → Reflect → Output

每个 Agent 内部都遵循这个循环（类 ReAct / Plan-and-Execute 框架）：

```
┌────────────────────────────────────────────────────────┐
│  ① Plan（规划）                                          │
│     LLM 分析任务 → 拆解为子步骤 → 选择工具               │
│     输出：[step_1: ..., step_2: ..., tool_chain: ...]   │
└──────────────────────┬─────────────────────────────────┘
                       ▼
┌────────────────────────────────────────────────────────┐
│  ② Tool（工具调用）                                      │
│     调用工具（数据采集 / 模型推理 / 数据库读写）          │
│     工具结果 → 短期记忆                                  │
└──────────────────────┬─────────────────────────────────┘
                       ▼
┌────────────────────────────────────────────────────────┐
│  ③ Observe（观察）                                       │
│     检查工具结果是否符合预期                              │
│     失败 → 回到 Plan 重新规划                            │
│     成功 → 进入 Reflect                                  │
└──────────────────────┬─────────────────────────────────┘
                       ▼
┌────────────────────────────────────────────────────────┐
│  ④ Reflect（反思）                                       │
│     LLM 自检：输出是否合 schema / 是否可操作 / 是否合规  │
│     有问题 → 修正 → 再 Reflect（最多 3 轮）              │
└──────────────────────┬─────────────────────────────────┘
                       ▼
┌────────────────────────────────────────────────────────┐
│  ⑤ Output（输出）                                        │
│     最终结果交付 → 写入长期记忆 → 触发反馈采集            │
└────────────────────────────────────────────────────────┘
```

**关键设计：**
- **Plan 不允许跳过**：禁止 LLM 直接 hallucinate 输出
- **Tool 调用有超时**：单 tool 调用 > 30 秒强制中断重试
- **Reflect 必须自检**：所有 LLM 输出强制 JSON schema 校验
- **Output 必须留痕**：完整 trace 日志（Plan + Tool + Reflect 全链路）

## 12.7 实战工作流走查（典型场景）

### 场景 1：用户提交一条新视频（PRO 用户）

```
T+0s   用户在小程序点击「提交诊断」，粘贴视频链接

         LLM 调用 Qwen3-32B（free）/ Qwen2.5-72B（pro 档位）→ 硅基流动

T+90s  渲染 H5 报告 → 写入数据库
T+91s  推送通知到小程序
T+92s  客户成功管家更新生命周期状态
T+5min 用户阅读报告 → 点👍👎 → 写入反馈池
       （周一进入 Prompt 迭代候选）
```

## 12.8 AI 成本与质量平衡（钱怎么花）

### 模型分层成本控制（核心降本策略）

| 档位 | 主力模型 | 单条 CT 报告成本 | 月度 ARPU | AI 成本占比 |
|---|---|---|---|---|
| 免费体验 | Qwen3-32B（硅基流动） | 0.1 元 | 0 | — |
| PRO 99 | Qwen2.5-72B / DeepSeek-V3.2（硅基流动） | 0.3 元 | 89 | 1.3%（4 条/月） |
| MAX 499 | DeepSeek-V3.2 / DeepSeek-R1（硅基流动） | 2 元 | 458 | 13%（30 条/月） |
| 加购（addon） | DeepSeek-V4-Flash + DeepSeek-R1（硅基流动） | 8 元/条 | 298 单价 | 3% |

**核心铁律：单 Agent 月度 AI 成本占该客户 ARPU 不得超过 15%。**

### 缓存与降本

| 优化 | 节省 |
|---|---|
| 头部账号数据 Redis 缓存（24h） | 减少 80% 重复爬取 |
| Few-shot 模板 prompt 复用 | 减少 30% token |
| 视频抽帧结果 OSS 缓存 | 减少 60% 重复抽帧 |
| 评分模型自研微调 | 替换 50% LLM 调用 |
| 批量请求合并 | 减少 20% API 费 |

### 质量监控

| 指标 | 阈值 | 触发动作 |
|---|---|---|
| 报告满意度 | < 80% | 启动 Prompt 紧急迭代 |
| Reflect 自检失败率 | > 10% | 升档使用更高级模型 |
| 用户投诉率 | > 1% | 暂停该 Agent，技术评估 |
| 单条诊断成本 | > 阈值 1.5 倍 | 立即切换降级模型 |

## 12.13 AI 安全与红线

| 红线 | 措施 |
|---|---|
| 不允许 AI 输出绝对化承诺 | Reflect 阶段强制合规扫描 |
| 不允许 AI 编造数据 | 所有数字必须来自工具调用真实结果 |
| 不允许 AI 输出违规内容 | 双层敏感词过滤（输入+输出） |
| 不允许 AI 处理私密信息 | 用户授权范围内严格使用，离开即销毁 |
| 不允许 AI 决策不可撤销操作 | 涉及付费/解约必须人工确认 |
| 不允许 AI 输出黑盒决策 | 每个判断必须给出推理链 |
